## 当前判断

1. 候选包有价值，但不是完整可直接覆盖的交付包。
2. 候选包中的 YOLOv5 letterbox、坐标反映射、tracking lineage、runtime replacement 状态表达有吸收价值。
3. 候选包仍存在缺口：不能证明真实目标视频上检测框问题已解决，也没有修完整 trust record 报告 hash 绑定。
4. 扫描和净化必须显式触发；full scan 默认自动净化已经被修正为不自动触发。
5. 检测框问题不能只看模型召回。坐标空间、时序复用、tracking hold、前端 overlay timeline 都可能造成“看起来像检测框错”的问题。
6. 提高推理分辨率可能改善远距离小目标召回，但在坐标和时序链路未闭环前，不能把分辨率当作第一解法。
7. 低分辨率训练模型不一定更好。更合理的方向是多尺度训练、贴近真实监控场景的数据增强，以及在坐标/时序修复后做 `640/960/1280` 对比实验。
8. 当前检测结果复用确实是风险来源之一；已加入 PPE-sensitive 更严格复用门限和 reuse diagnostics，并已完成第一轮前后端 timeline lineage 收紧。
9. 当前 `overlay_coordinate_space.preview_frame_shape` 表示 runtime/preview 帧尺寸，不等于浏览器 Web canvas 尺寸；前端 canvas 缩放仍需独立闭环。
10. `ppe_small_target_diagnostics` 只是诊断字段，不等于已经启用低 context 小目标放行策略。
11. `prepare_runtime_for_start(auto_remediate=True)` 仍是 service 内部保留分支；生产 `/api/start` 必须固定 `auto_remediate=False`。
12. `report_path` 当前只能视为已存储路径，不能视为已完成 report identity/hash 绑定校验。
13. 自定义 ONNX、TensorRT 等加速 runtime artifact 必须显式提供同源 source PT，不能猜测同目录 `best.pt`。
