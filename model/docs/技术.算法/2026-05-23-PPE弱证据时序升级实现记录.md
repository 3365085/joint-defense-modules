# PPE 弱证据时序升级实现记录

## 背景

用户要求按优化路线落地 PPE 检测优化：不改模型，不增加主路径额外推理，优先解决远处小目标 `head` 被降级为 weak 后只显示、不进入有效 PPE 判定的问题，同时补上 `helmet` 的同类时序逻辑。

## 实现结论

1. 已在 PPE 后处理链路增加 `weak_helmet_indices`，与既有 `weak_head_indices` 对称。
2. 已在 `PPEDisplayTracker` 中增加 weak 证据连续命中状态：
   - `weak_head_streak`
   - `weak_helmet_streak`
   - `promoted_label`
   - `temporal_promoted`
3. 稳定 `weakHead` 会在当前帧升级为有效 `head`，进而进入 `missing_helmet_count` 和 PPE candidate 判定。
4. 稳定 `weakHelmet` 会在当前帧升级为有效 `helmet` 安全证据，但低置信 isolated helmet 必须具备当前 `head/person` 上下文；否则需要达到更高平均置信度，避免背景区域低置信 helmet 被误升为安全证据。
5. weak 目标仍不允许缺帧后长时间 hold 成有效证据，避免拖框和旧框滞留。

## 代码链路依据

- `model/src/defense/module_a/ppe_postprocess.py::suppress_helmet_false_positives` 输出 `weak_helmet_indices`。
- `model/src/defense/module_a/postprocess/ppe_tracking.py::PPEDisplayTracker` 维护 weak streak、时序升级和 helmet 上下文约束。
- `model/src/defense/runtime/ppe_business.py::evaluate_ppe_business` 先更新 tracker，再调用 `apply_temporal_evidence`，最后进入 `SafetyHelmetState.update`。
- `model/src/defense/runtime/frame_processor.py::FrameProcessor._build_status` 输出 `ppe_weak_helmet_count`、`ppe_promoted_head_count`、`ppe_promoted_helmet_count`、`ppe_effective_head_count`、`ppe_effective_helmet_count`。
- `model/src/defense/visualization/overlay.py::draw_ppe_boxes` 将 promoted 目标显示为 `head+` / `helmet+`，未升级弱证据显示为 `weakHead` / `weakHelmet`。

## 关键参数

- `ppe_tracking.weak_promotion_hits: 3`
- `ppe_tracking.weak_head_min_avg_confidence: 0.30`
- `ppe_tracking.weak_helmet_min_avg_confidence: 0.30`
- `ppe_tracking.weak_helmet_isolated_min_avg_confidence: 0.50`
- `ppe_tracking.weak_edge_promotion_hits: 4`
- `ppe_tracking.weak_edge_min_avg_confidence: 0.45`
