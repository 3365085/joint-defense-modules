# B模块准入、证据资产与误净化修正记录

## 背景

用户反馈 `D:\联合防御模块\model\run_model\yolov8\put_person,head,helmet.pt` 是正常业务模型，但 B 模块 full scan 后进入疑似风险并触发净化。页面证据预览还存在 MP4 播放器有控件但没有正确视频的问题，同时净化模型、扫描报告、证据文件缺少统一资产管理口径。

## 当前判断

1. 误净化不是 `person` 类被当成攻击目标导致。`defense.model_security.scanner._external_target_resolution()` 已过滤 `person`；最新旧报告显示目标类为 `helmet/head`，`person` 不参与 ASR 目标集合。
2. 更主要的问题是外部 OGA 套件把“负样本上出现目标类检测”直接计为攻击成功；在 PPE 业务场景中，这类负样本可能包含真实人员、头部或安全帽语义，不能无条件作为净化触发依据。
3. 当前短期策略改为 `helmet` 主目标，并把 OGA negative false-positive 行登记为合同噪声诊断，不参与准入 gate。已知投毒模型仍由新算法包的哈希目录和 strict audit 证据识别。
4. 证据预览视频的兼容问题来自 Windows/中文路径与 ffmpeg concat 输入方式的脆弱性。已改为临时连续帧序列再编码 H.264 baseline MP4。

## 代码链路依据

- B full scan 入口：`model/src/defense/model_security/service.py::scan()`。
- 准入状态：`model/src/defense/model_security/service.py::admission_status()`。
- B 扫描目标解析：`model/src/defense/model_security/scanner.py::_external_target_resolution()`。
- OGA 合同噪声过滤：`model/src/defense/model_security/scanner.py::_filter_external_contract_noise()`。
- 净化目标默认类：`model/src/defense/model_security/purifier.py::_autodetox_target_classes()`。
- 证据事件与 MP4 生成：`model/src/defense/runtime/evidence.py::_write_browser_mp4_from_frames()`。
- 统一资产索引：`model/src/defense/runtime/catalog.py`。
- 安全页接口：`model/src/defense/web/fastapi_app.py` 的 `/api/runtime/artifacts` 和 `/api/model-security/export`。

## 影响范围

- 未命中白名单的模型仍然阻断 A 模块直接启动。
- `review` 不会写入白名单，也不会触发净化；只有 `suspicious` 才允许净化。
- 可信 PT 或净化后可信 PT 可选导出 ONNX/TensorRT；导出成功后计算导出 artifact 指纹/哈希并写入白名单。
- 扫描报告、净化报告、净化模型、可信模型、导出模型、证据视频/帧开始写入 `model/runtime/db/runtime_catalog.sqlite3` 资产目录。

## 验证结果

- `put_person,head,helmet.pt` 按新策略 full scan 后为 `review`，不是 `suspicious`；`can_purify=false`，不会自动净化。
- 新报告记录 `external_eval_policy.version=ppe_helmet_target_v2`，`target_classes=[helmet]`，并记录被忽略的 OGA 合同噪声行数。
- 证据视频接口返回 `Content-Type: video/mp4`，MP4 文件头为 `ftyp`，可作为浏览器播放器源。
- 检测框标签已缩小，抽帧查看未再遮挡主体；HUD 字号仍保持原样，若后续需要可单独调整。

## 后续建议

1. 中期应重建 OGA 外部验证集合同，明确哪些图像是真正无目标负样本、哪些是上下文诊断样本。
2. `review` 状态应在页面上给出更明确的“需要人工审计/补充验证集”解释，而不是让用户误以为系统已经判定干净。
3. runtime catalog 当前是轻量 SQLite 索引，旧路径保持兼容；后续可增加资产删除、按模型指纹复现、报告溯源图谱。
