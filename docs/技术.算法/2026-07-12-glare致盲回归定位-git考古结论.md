- **glare 008/016 掉零不是某次 commit 引入的回归**。detector.py 全历史仅两次提交
  (`129ce9c` 移植、`e270e4d` 联合决策修复), 逐一实测: **两个版本都 MISS glare 008/016**。

  该状态在 7-12 整理入库时被后续改动覆盖, git 中不存在对应快照。

## glare 为何 MISS —— 逐帧诊断(当前代码, clip_0061=glare-008)

glare 是致盲型攻击, 设计确认路径 = **支路B(blind branch, p_blind)**。实测该支路信号被自身机制掐死:

- glare 使 YOLO 从第 6 帧起丢目标(全片 no-ROI 帧 78/151; glare-016 更极端 172/180)。
- 支路B 融合式: `evidence = max(target_loss, 0.5*glare_blind)`, `p_blind = degrade*(0.35+0.65*evidence)`。
- **两个致盲证据都被自身逻辑抹掉**:
  1. `target_loss = det_drop if (sum(recent_target_presence)>=2) else 0`。
     `recent_target_presence` 是 maxlen=8 的 deque, 目标丢失后 8 帧内耗尽 → 第 12 帧起 sum<2 → target_loss≡0。
  2. `det_drop = (ref_det - det_strength)/ref_det`, 而 `ref_det` 来自场景基线 `_sb_detstr` 的 p70。
     glare 段逐帧喂入 det_strength=0(无检出)进基线 → ref_det 从 1.54 一路衰减到 0(第 21 帧) → det_drop≡0。
- 结果: 致盲段只剩 `glare_blind≈0.45-0.53` 单独扛, `p_blind` 封顶 ≈0.25-0.27, 远低于 `theta_blind=0.55`。
  全片 p_blind≥0.55 仅 4 帧(016 仅 1 帧), 而 `blind_hit_required`≈6(0.6×window) → **blind 永不确认**。
- adv 支路虽 p_adv 高(008: 70 帧≥0.65)但候选被 `scene_baseline_normal` + `target_related_feature` 失败
  打散(adv_count 峰值 4-5 < adv_hit_required=6); sustained 升级 floor=120 帧(process_fps 学到~60×2s)
  在 151 帧短片上不可达。→ adv 支路也确认不了。

注: 支路B 的 `_compute_blinding`/`blind_hit_required` 在 demo-6-30 / 129ce9c / 当前**三处字节一致**,
`recent_target_presence` maxlen=8 三处也一致。故 glare 从来就靠这套逻辑, 而这套逻辑对"目标全程丢失+
基线学零"的强致盲天然失效。7-9 能命中, 只可能是当时中间态的**其它参数/逻辑组合**(未入库)恰好托住了它。

## 恢复可行性判断

- **无法靠回退 commit 恢复**: 7-9 的命中态无对应提交, git 中不存在。
- **7-9 的 sustained ratio(固定100窗/0.68)也不是 glare 的通路**: 仿真 100 窗原始触发占比,
  glare-008=0.48 / glare-016=0.53 均 <0.68 够不到升级; 反而 clean-015 占比=1.0 会误升级。
  即"恢复旧 sustained"既救不了 glare 又会引入误报。
- **可行的修复方向(待实验确认, 非本轮改)**: 让致盲证据不被自身抹掉 ——
  (a) target_loss 的 `recent_targets` 门用"曾稳定有目标"的**长记忆/锁存**替代 maxlen=8 的滑窗,
      使目标被持续致盲时 target_loss 不衰减为 0;
  (b) 场景基线更新排除 det_strength=0 的疑似致盲帧(类似 C1 对 A1 的豁免思路), 防 ref_det 被学成 0;
  (c) 或对"曾有目标 + 当前 no-ROI + glare_blind 持续高"给 p_blind 一个不依赖 ref_det 的兜底。
  三者都要双口径验证(守召回≥90.5% / 误报≤7.4%), 尤其防 016 干净段与 015 的近阈值误报恶化。
