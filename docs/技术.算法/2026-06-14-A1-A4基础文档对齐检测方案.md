# A1-A4 基础文档对齐检测方案

日期：2026-06-14

## 参考边界

本方案严格参照 `D:\联合防御模块\doc\前置文档\基础文档.txt` 中模块 A 的定义：

- A1：单帧纹理异常检测，使用 LBP 计算局部纹理与全局纹理差异 ΔH。
- A2：时序纹理不一致性分析，计算相邻帧之间 LBP 特征图变化率 Change_t。
- A3：光流伪影检测，统计光流场局部异常区域并提取 f_flow。
- A4：融合分类，将 ΔH、Change_t、f_flow 拼接后输入轻量二分类器，输出 p_adv。

本方案不重新定义 A1-A4，也不把强光、模糊、遮挡、贴片分别改造成新的 A1/A3/A3b 模块。新增机制只放在 A4 输出之后，用于降低正常视频误报、保证连续帧报警、增强解释性。

## 当前问题

当前效果不佳的根因可以概括为：A4 的单帧 p_adv 输出没有经过足够的安防场景约束。正常视频中，摄像头移动、自然曝光、行人正常运动、背景边缘变化都可能导致 ΔH、Change_t 或 f_flow 偏高；如果 A4 单帧高分直接进入报警或显示层，就容易出现正常视频全程报警。

同时，攻击视频需要可解释：用户需要看到每个视频主要是哪一路证据导致报警，而不是只有一个不透明的 p_adv。因此需要在 A4 后增加解释层，但解释层只能解释 A1-A3 对 A4 的贡献，不能改变基础链路。

## 总体架构

处理链路保持为：

```text
视频帧 Ft
  -> A1: LBP 单帧纹理异常 ΔH
  -> A2: LBP 时序纹理变化 Change_t
  -> A3: 光流伪影特征 f_flow
  -> A4: 融合分类 p_adv
  -> A4 后处理: 场景门控 + 短窗确认 + 结果解释
  -> single_frame_suspicious / alert_confirmed / public_reason
```

其中 A4 前半段只负责“算法检测”，A4 后处理负责“安防语义确认”。

## A1-A4 输出约定

### A1 输出

- `delta_h_global`：全局 LBP 纹理差异。
- `delta_h_local_max`：局部最高纹理异常。
- `delta_h_roi_max`：目标区域或候选区域内最高纹理异常。
- `a1_score`：由 ΔH 归一化得到，仅用于 A4 输入和解释。

A1 不直接报警。

### A2 输出

- `change_t`：相邻帧 LBP 特征图变化率。
- `change_t_local_max`：局部时序纹理突变。
- `change_t_without_motion_target`：是否出现“纹理突变但无对应运动目标”。
- `a2_score`：由 Change_t 归一化得到，仅用于 A4 输入和解释。

A2 不直接报警，但可以作为动态扰动解释依据。

### A3 输出

- `flow_score`：光流异常综合分数。
- `flow_local_ratio`：光流异常局部集中度。
- `flow_shape_score`：异常区域形状是否不符合正常运动。
- `flow_target_relation`：异常光流是否与目标区域或候选区域存在空间关系。
- `a3_score`：由 f_flow 归一化得到，仅用于 A4 输入和解释。

A3 不直接报警。

### A4 输出

- `p_adv_raw`：轻量二分类器原始输出。
- `p_adv_calibrated`：经过基础校准后的单帧攻击概率。
- `feature_contrib`：A1/A2/A3 对 p_adv 的贡献估计。

A4 的 `p_adv_calibrated > θ_adv` 只表示“单帧候选异常”，不等价于最终告警。

## A4 后处理机制

### 1. 场景门控

场景门控接收 A4 输出和当前视频上下文，只决定是否允许单帧候选进入短窗确认。

允许进入短窗的条件：

- `p_adv_calibrated >= θ_adv`；
- A1/A2/A3 中至少一路达到有效贡献阈值；
- 异常不是明显自然场景变化；
- 若存在目标框，则异常区域应尽量与目标框或目标附近区域相关；
- 若无目标框，只允许极端情况或“前几帧有稳定目标、当前目标突然消失”的兜底逻辑。

抑制条件：

- 全局相机平移或抖动导致的均匀光流；
- 室内到室外、逆光、自动曝光造成的整体纹理/亮度变化；
- 背景广告牌、柱子、门框、窗框等固定结构造成的纹理异常；
- 行人正常运动导致的光流变化，但没有 A1/A2 支持；
- 单帧孤立高分，前后帧无连续性。

输出：

- `candidate_allowed: bool`
- `gate_reason: str`
- `suppressed_reason: str`

### 2. 短窗确认

严格沿用基础文档的连续帧思想：单帧只标记，连续多帧才实时告警。

建议策略：

- 默认：5 帧中 3 帧通过门控，则 `alert_confirmed=True`。
- 高误报场景：6 帧中 4 帧通过门控。
- 低延迟场景：4 帧中 3 帧通过门控。

短窗统计对象不是原始 `p_adv_raw`，而是 `candidate_allowed=True` 的候选帧。这样可以避免正常视频中很多单帧纹理/光流噪声堆出报警。

### 3. 结果解释

解释层不重新定义模块，只解释 A4 的输入贡献。

对每个报警输出：

- `dominant_input`: `A1_LBP_SINGLE` / `A2_LBP_TEMPORAL` / `A3_FLOW_ARTIFACT` / `A4_FUSION`
- `public_reason`: 面向用户的短原因。
- `debug_reason`: 详细诊断原因。
- `a1_display_score`
- `a2_display_score`
- `a3_display_score`
- `a4_display_score`

显示分数规则：

- A1/A2/A3 显示分数来自它们对 A4 的贡献和归一化强度。
- A4 显示分数来自 `p_adv_calibrated` 与短窗确认状态。
- 没有通过场景门控的帧，显示分数应被压低，不应在 UI 上表现为攻击。

## 攻击视频解释目标

该方案不预设新的攻击类别，但可以通过贡献解释满足“每个视频尽量一个模块报警值高”的需求：

- 闪烁光照、动态光照扰动：通常 A2 贡献高，A4 确认；如果单帧纹理差异明显，A1 辅助升高。
- 移动对抗贴片：A2 与 A3 贡献高，A4 确认；A3 光流局部集中度应成为主要解释。
- 动态遮挡：A3 贡献高，A2 辅助，A4 确认。
- 纹理型对抗衣物/贴纸：A1 局部 LBP 异常贡献高，A3 若随目标运动则辅助。
- 普通正常视频：A1/A2/A3 可以有诊断波动，但 A4 后门控和短窗不应确认。

## 与当前代码的落点

建议保留现有 A1-A3 特征计算代码和 A4 融合入口，优先调整以下边界：

1. 在 `process_pipeline.py` 中保留原始 `fusion.compute(...)` 输出，新增 A4 后处理步骤。
2. 在 `target_anchored.py` 或新的 `post_fusion_gate.py` 中实现场景门控，但命名上应表达为 A4 后处理，不应改写 A1-A4 定义。
3. 在 `alert_state.py` 中继续使用 3/5 或可配置短窗确认。
4. 在 `detail_builders.py` 中拆分 `debug_reason` 与 `public_reason`。
5. 在 `rebuilt_demo` 中删除 `single_frame_suspicious -> p_adv=0.85` 的展示放大逻辑，改为展示 A4 后处理后的确认结果和解释分数。

## 验收标准

正常视频：

- `室内到室外视频.mp4`：`alert_confirmed` 应为 0。
- `固定镜头室外视频.mp4`：`alert_confirmed` 应为 0。
- 单帧 `p_adv_raw` 可以波动，但 `candidate_allowed` 和 `alert_confirmed` 应受门控约束。

攻击视频：

- 攻击出现后，A4 后处理应在短窗内确认。
- 每个视频应输出主要贡献输入：A1、A2、A3 或 A4。
- 至少一个显示分数应在攻击区间明显升高，而正常区间保持低位。

## 结论

基础文档的核心链路应保持为 LBP 单帧纹理异常、LBP 时序纹理不一致、光流伪影检测、融合分类报警。工程修订不应重新命名或重构 A1-A4 的职责，而应在 A4 输出之后增加安防场景语义：先门控，再短窗确认，最后解释 A1/A2/A3 对 A4 的贡献。这能兼顾基础文档一致性、正常视频低误报和攻击视频可解释报警。
