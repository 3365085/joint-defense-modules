## 问题背景

本轮复核针对 `oga_visible_patch` 与 `oga_semantic_vest` 两个实验的 `clean / attack / purif` 三栏检测对比。前序问题集中在四类风险：

- `oga_visible_patch` 贴纸样式偏离原始定义，曾出现青紫棋盘格替代红色矩形 + 黄色叉的问题。
- `oga_visible_patch` 最终展示不应基于 `attack_eval_sequence` 图片序列，而应基于真实视频帧。
- clean 列必须使用用户认可的三类 PPE 基线模型，不能沿用旧 `baseline_yolov8` 路径。
- `oga_semantic_vest` 数据集与验收不能只关注 `head -> helmet`，必须把目标人物 `person` 框纳入硬验收，避免 person 类在三栏中不稳定。
