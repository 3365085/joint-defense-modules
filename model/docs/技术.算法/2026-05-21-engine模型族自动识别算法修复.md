# Engine 模型族自动识别算法修复记录

## 问题背景

用户确认当前阶段不修模型权重，先按 `best.engine` 修检测算法。原项目中 `best.engine` 存在 YOLOv5 TensorRT 与 YOLOv8/Ultralytics TensorRT 两类同名产物，二者输出张量格式不同，不能只凭 `.engine` 后缀统一走同一个后端。

## 当前判断

当前检测算法不必须依赖 `person` 类。PPE 业务判断可以由 `head` 裸头证据触发，也可以由 `person` 与有效 `helmet` 数量关系触发。`person` 类主要提供人员上下文与人数约束，不应成为 `best.engine` 是否能正常检测的唯一前提。

检测效果主要由以下因素共同决定：

- 模型产物本身的类别表、训练数据和导出格式。
- 推理后端是否按正确模型族解码输出张量。
- 推理阈值、NMS、输入尺寸、BGR/RGB 与缩放预处理。
- PPE 后处理对 `person`、`helmet`、`head` 的上下文规则。
- 跟踪、状态保持、ROI 复检和业务告警窗口。

## 代码链路依据

- `src/defense/runtime/config.py`
  - `infer_backend_from_model_path()` 根据 `.engine` 选择 TensorRT 后端。
  - `infer_model_family_from_model_path()` 负责为自定义模型选择 `yolov5` 或 `ultralytics` 模型族。
  - `apply_custom_model()` 将自动识别结果写入 `inference.model_family` 与 `inference.artifacts.engine`。
- `src/defense/module_a/backends/detector_backend.py`
  - `YoloV5DetectorBackend` 按 YOLOv5 TensorRT 输出格式自行做预处理、NMS 和类别解码。
  - `UltralyticsDetectorBackend` 交给 Ultralytics 运行时处理 YOLOv8/Ultralytics engine。
- `src/defense/module_a/ppe_postprocess.py`
  - `summarize_ppe_from_detections()` 汇总 `person_count`、`helmet_count`、`head_count`、`missing_helmet_count`。
  - `suppress_helmet_false_positives()` 使用人员/头部上下文抑制安全帽误检。

## 修复内容

- 增加 `.engine` 模型族识别：
  - 优先用路径中的 `yolov5`、`yolov8`、`yolo11`、`ultralytics` 明确线索判断。
  - 没有路径线索时，尝试读取 TensorRT engine 输出维度判断：
    - `[1, anchors, classes+5]` 形态判为 `yolov5`。
    - `[1, classes+4, anchors]` 形态判为 `ultralytics`。
- 增加回归测试，覆盖：
  - `baseline_yolov5/.../best.engine` 自动选择 `yolov5`。
  - `baseline_yolov8/.../best.engine` 自动选择 `ultralytics`。
  - 自定义 YOLOv5 engine 自动写入 `tensorrt + yolov5`。

## 影响范围

该修复只影响自定义 `.engine` 的自动模型族选择，不改变模型权重、类别语义、置信度阈值、PPE 判定阈值或 Web API 字段。默认配置中已经显式声明 `inference.model_family: yolov5` 的场景不受影响。

## 结论

当前算法不一定要使用 `person` 类，但必须按 engine 的真实模型族选择正确解码链路。否则同名 `best.engine` 会被错误后端解释，表现为类别错乱、空检或检测稀疏。此次修复优先解决 `.engine` 自动路由错误。

## 后续建议

- 继续检查 PPE 文案和状态字段，避免把“没有检测到 person 类”误写成“画面没有人员”。
- 对 `best.engine` 增加视频级回归样本，固定每秒抽帧的 raw boxes、业务 PPE 状态和显示 tracks。
- 如仍出现检测稀疏，再单独比较原项目启动参数、输入预处理、TensorRT/Ultralytics 版本和视频抽帧链路。

- 仅从代码无法确认用户口中的“原本项目没问题”具体对应 `baseline_yolov5` 还是 `baseline_yolov8` 的 `best.engine`。
- 仅从代码无法确认目标视频中每一帧的真实标注，因此检测效果好坏仍需结合人工标注或原项目输出视频做实验对照。
