# 交付包可移植运行数据目录

日期：2026-07-18

## 背景

交付包可能被解压到任意盘符和任意目录。外部 Pixi 环境只提供 Python 库，不应决定源码、模型或运行数据位置。若数据库、evidence、B 模块白名单、缓存和日志继续写入交付目录，首次运行会改变包哈希，也会把测试状态带入重新打包结果。

## 设计

- 新增统一环境变量：`DEFENSE_RUNTIME_DATA_ROOT`。
- 主项目未设置该变量时，继续使用项目内 `model/runtime`，保持现有开发环境兼容。
- 交付启动器默认设置为 `%LOCALAPPDATA%\JointDefense\runtime`。
- 用户可以在启动前把 `DEFENSE_RUNTIME_DATA_ROOT` 设置为其他绝对路径。
- 交付启动器拒绝把运行数据根目录设回交付项目内部。
- `PYTHONDONTWRITEBYTECODE=1` 阻止在包内生成 `__pycache__` 和 `.pyc`。

## 路径归属

写入统一运行根目录：

- `db/runtime_catalog.sqlite3`
- `evidence/monitor`
- `model_security`：白名单、完整性 seal、扫描报告、净化输出、导出模型和缓存
- `artifacts/video_decode`：NVDEC 派生视频缓存
- `logs/yolov5_runtime`：Ultralytics/YOLOv5 配置与缓存

非 ASCII 视频解码别名需要与源视频同盘才能保持硬链接。源视频与 LocalAppData 不同盘时，临时别名改用源盘 ASCII 目录 `X:\module_a_runtime_alias\...\runtime\video_decode_alias`；退出后按原生命周期清理。

继续保留在交付包内，只读使用：

- `model/runtime/artifacts/yolo`
- `model/runtime/artifacts/module_a`
- 权威 PT、ONNX、TensorRT engine、RAFT engine、A4 分类器及元数据

## 影响

- 任意解压路径均可启动。
- 运行后交付源码、模型和 SHA-256 清单不应变化。
- 不同 Windows 用户拥有独立运行状态。
- 删除 `%LOCALAPPDATA%\JointDefense\runtime` 会清空本机 evidence、数据库和 B 白名单；不会删除交付模型。
- 多个交付版本默认共享同一用户运行目录。若需版本隔离，应在启动前设置不同的 `DEFENSE_RUNTIME_DATA_ROOT`。
- B 独立算法源码保持逐文件不变；联合运行层 `defense/model_security/service.py` 只接收独立 `runtime_root`，扫描和净化算法不变。

## 验证

- 主项目和交付包统一路径契约测试通过。
- evidence、catalog、B 存储、NVDEC 派生缓存、解码别名和 YOLO 配置路径均已覆盖。
- 交付 GPU `VerifyOnly` 必须显示实际 `runtime_data_root`，并拒绝包内路径。
