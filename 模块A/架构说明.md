# 模块 A 实时视频防御系统 — 技术路线与功能检测路线

## 一、系统定位

本系统是一条 **GPU-first 的视频物理对抗扰动 / 翻拍假目标实时检测管线**，面向施工安全、周界监控等场景，核心任务是：

- **物理对抗扰动（p_adv）**：检测画面中是否出现破坏目标检测的物理攻击（强光、可见性退化、运动模糊、遮挡、对抗补丁）
- **翻拍/假目标（A3b）**：检测画面中是否出现用手机屏、纸张照片、平板/显示器翻拍的假目标
- **伪造视频源（p_synth）**：检测输入是否为 AI 生成 / 深度伪造 / 回放视频（**当前交付版本代码层面禁用，仍在开发中**）
- **安全帽业务（p_safety）**：业务侧独立通道，判断人员是否戴安全帽（沿用既有能力，本系统不修改）

## 二、整体技术路线

### 2.1 端到端管线

```
视频输入（MP4 / RTSP / 摄像头）
    │
    ▼
StreamSource（后台拉流 + 显式丢帧队列）
    │
    ▼
resize → 640×640
    │
    ▼
DetectorBackend（YOLOv5 v1，TensorRT/ONNX/PyTorch）
    │
    ├─ boxes / classes / confidences（helmet, head, person）
    ▼
ROIProvider（按置信度过滤 + 边距）
    │
    ▼
ModuleADetector（A1–A3b 特征 + 源真实性）
    │
    ├─ A1 过曝                    → ratio, is_glare
    ├─ A2 LBP 纹理 + 时域纹理      → delta_h, change_t, local_max
    ├─ A3 运动/模糊/跟踪/光流     → motion_score, blur_score, track_score, flow_score
    ├─ A3b 翻拍/假目标检测         → static_media.score, static_media.triggered, p_media
    └─ source_authenticity        → p_synth (开发中，默认关闭)
    │
    ▼
Fusion（5 维线性加权 + A4 分类器双通道取 max）
    │
    ├─ p_adv = max( Σ wᵢ·featureᵢ, A4_classifier )
    │   其中 feature = [delta_h, change_t, motion_score, overexposure_ratio, static_image_score]
    │   权重 = [0.20, 0.30, 0.20, 0.10, 0.20]
    │
    ▼
AlertState（3/5 连续帧确认 + 时间窗约束）
    │
    ▼
ModuleAResult → Web 监控台三路告警卡片 + 事件证据归档
```

### 2.2 关键设计决策

| 决策 | 原因 |
|------|------|
| **GPU-first，不允许静默 CPU fallback** | 性能必须满足实时（≤ 15 ms/frame @ 640×640），CPU 无法达标 |
| **YOLOv5 v1 作为主线** | 重新训练三类模型（helmet/head/person），针对目标场景优化 |
| **三路告警分路契约** | p_adv / p_safety / p_synth 不得合并为单分数，否则一路误报淹没另一路 |
| **A3b 通过 rule_fusion 第 5 维融入 p_adv** | 符合 Req 7.3 "不得新增独立告警路径"，同时让 p_adv 真正体现综合风险 |
| **p_adv 取 rule_fusion 和 A4 分类器的 max** | 既保留规则可解释性，又吸收分类器对复杂场景的拟合能力 |
| **3/5 连续帧确认** | 避免单帧抖动误报，平衡实时性和稳定性 |

## 三、A1–A3b 特征检测路线

### A1：过曝检测（`overexposure.py`）

- **目标**：检测强光攻击（glare）造成的过曝区域
- **方法**：灰度图素直方图 + 高亮区域连通域分析
- **输出**：`ratio`（过曝像素比）、`is_glare`（布尔标志）

### A2：纹理异常

**LBP 纹理分析（`lbp_texture.py`）**
- 计算全图 LBP 编码直方图
- 与前一帧直方图对比，得到 `delta_h`（纹理变化强度）

**时域纹理分析（`temporal_texture.py`）**
- ROI 内部帧间差异，得到 `change_t`
- 局部最大响应 `local_max`
- 触发 `temporal_texture_change` / `local_temporal_texture_change` reason codes

### A3：运动/模糊/跟踪/光流

**motion_artifact.py（运动伪影）**
- 帧间差分 + 光流异常区域比例
- 输出 `motion_score`、`local_max_ratio`

**blur_degradation.py（模糊退化）**
- 局部频谱能量比例（Laplacian 方差）
- 输出 `blur_score`、`blur_low_energy_ratio`
- 触发 `blur` reason code（对应 motion_blur 攻击）

**track_consistency.py（轨迹一致性）**
- 跨帧 ROI ID 稳定性追踪
- 输出 `track_score`、`track_drop_score`、`confidence_drop_score`
- 触发 `track_consistency_drop`（对应 occlusion 攻击）

**light_flow.py（轻量光流）**
- 周期性执行的低成本光流异常检测
- 输出 `light_flow_score`、`light_flow_local_anomaly_ratio`

### A3b：翻拍/假目标检测（`features/a3/static_media/`）

A3b 有两条并行路径，OR 融合后输出：

**Legacy 路径（patch-track）**
- 对每个 YOLO ROI 做 patch 相似度比较
- 触发条件：`similarity ≥ 0.97 AND stable_count ≥ 6 AND motion_evidence AND edge_mean ≥ 阈值`
- 针对真人缓慢移动做了严格 gate，避免误报

**A3+ 路径（L0→L1→L2→L3）**
- L0：Canny 边缘 + 轮廓提取候选矩形区域
- L1：IoU 跟踪（CandidateTrackManager）
- L2：单应性验证（HomographyVerifier，每 3 帧）
- L3：p_media 决策（3 种类型：static_image / screen_replay / embedded_video）
- 触发条件：`p_media ≥ 0.65 AND track_score ≥ 0.7`

任一路径触发 → `static_image_triggered=True` + `static_image_score` 被提升

## 四、A4 融合层

### 4.1 Rule Fusion（5 维线性加权）

```python
feature = [
    delta_h,              # A2 纹理变化
    change_t,             # A2 时域纹理
    motion_score,         # A3 运动伪影
    overexposure_ratio,   # A1 过曝
    static_image_score,   # A3b 翻拍
]
weights = [0.20, 0.30, 0.20, 0.10, 0.20]
p_adv_rule = clamp(dot(feature, weights), 0, 1)
```

### 4.2 A4 分类器（TorchLogisticFusion）

- 独立的 logistic/MLP 模型，消费 46 维特征（包括所有 A1–A3b 原始分数 + 规则触发标志）
- artifact：`experiments/configs/module_a_classifier_universal_yolov5_yolov8_v1.json`
- **通用校准**：YOLOv5 v1 和 YOLOv8 v2 共用同一个 A4 分类器

### 4.3 最终 p_adv

```python
p_adv = max(p_adv_rule, classifier_p_adv)
```

这一步确保：
- A3b 通过 rule_fusion 贡献的分数不会被 classifier 压 0
- A4 分类器对复杂模式的拟合能力被保留
- p_adv 数值 ≠ A3b 原始 score（两个卡片显示不同数字）

## 五、告警状态机（AlertState）

- **离线模式**（纯帧计数）：5 帧滑动窗口中 3 帧 `is_suspicious=True` → `alert_confirmed=True`
- **在线模式**（带时间戳）：除了帧计数，还要求窗口内时间跨度符合实时约束（抵抗抽帧抖动）
- **Hold 机制**：确认后继续 hold N 帧，避免闪断

## 六、Web 监控台（`tools/module_a_monitor_app.py`）

- 本地 Flask 服务，默认 `http://127.0.0.1:7860/`
- 三路告警卡片分路展示：**物理对抗扰动 (p_adv)** / **翻拍/假目标检测 (A3b)** / **视频源真实性 (p_synth)**
- 业务告警独立区域：安全帽 (PPE)、模块 A 扰动告警事件列表、生成/伪造视频流提示（当前关闭）
- **伪造视频流告警（p_synth）在代码层面强制关闭**，UI checkbox 置灰并标注"开发中"
- 支持输入：MP4 文件、RTSP/HTTP 视频流、本机摄像头编号
- 异常事件自动归档到 `异常记录/p_adv/<source>/<event>.json` + 短视频 + 代表帧

## 八、运行约定

- CUDA 是硬约束，不允许静默 CPU fallback
- YOLO 推理优先用 TensorRT `.engine`，跨设备/边缘场景用 `.onnx` 重新构建
- 主配置 `experiments/configs/module_a_baseline.yaml`
- A4 分类器 artifact 是 JSON 格式的轻量模型权重
- 所有新代码进入 `defense/module_a/` 或 `defense/pipelines/`，不引入旧 layer/semantic recovery 设计
