# B模块净化指标与测试绕过入口技术交底

## 背景

白名单页面此前只显示净化后模型的运行路径和风险值，容易造成两个误解：

1. 以为净化完成后仍在直接运行原始投毒模型。
2. 把 `0.004` 这类风险值误读为原始模型 ASR。

实际链路中，`0.004` 来自净化后 strict audit 的 Wilson 95% 上界，表示在严格验证集上观察到 0 次成功攻击时的统计上界，不是原始投毒模型 ASR。原始投毒模型的 ASR 需要单独从已知毒模型证据或外部验证报告中读取。

## 代码链路依据

- `defense.model_security.scanner.full_scan`：执行 full scan，已知投毒包命中时返回 `status=suspicious`、`risk_score=1.0`；strict purified 模型命中时使用 strict audit 指标作为风险依据。
- `defense.model_security.purifier.packaged_poisoned_evidence_for_model`：按模型 hash 识别新算法包里的 poisoned 模型，并关联对应 strict purified 候选。
- `defense.model_security.purifier.known_poisoned_attack_metrics`：返回已知 poisoned 家族的原始攻击 ASR、成功数、样本数和证据来源。
- `defense.model_security.service.ModelSecurityService.purify`：可疑模型净化后，对候选净化模型再次执行 full scan；只有 clean/trusted 结果才写入白名单。
- `defense.model_security.service.ModelSecurityService._security_metrics`：汇总原始 ASR、净化后 ASR、Wilson 上界、mAP drop、净化报告路径等指标。
- `defense.model_security.registry.TrustRecord`：白名单记录以实际运行模型 hash、类别语义 hash、PPE 语义 hash 和安全指标为准。
- `defense.web.fastapi_app.create_app` 中的 `/api/start`：A模块启动前执行 B 准入；测试绕过字段只影响本次启动请求，不写入白名单。

## 本版优化

1. 白名单记录新增 `original_source_model_path` 和 `original_source_model_hash`，用于说明净化链路的原始输入模型。
2. 白名单记录新增 `security_metrics`，包含原始 ASR、净化后 ASR、净化后 Wilson 上界、mAP drop、审计来源、净化报告路径等字段。
3. 安全中心状态卡新增“原始ASR / 净化后ASR / 净化后Wilson上界”，避免把净化后统计上界误读成原始风险。
4. 白名单卡片显示“原始源模型”和“ASR对比”，可直接看出原始模型与净化后模型的关系。
5. 新增临时测试入口 `test_bypass_model_security`，允许当前研发阶段绕过 B 准入，直接运行任意模型做检测效果对比。

## 测试绕过入口边界

- 只用于本地研发测试，页面文案明确标注“不会写入白名单，禁止生产使用”。
- 不会调用 `ModelTrustRegistry.mark_trusted`。
- 不会修改扫描报告、净化报告或模型安全状态。
- `/api/start` 响应中的模型安全状态会标记为 `bypassed_for_test`。
- 生产策略仍然是：白名单命中、full scan clean，或 suspicious 经净化并复扫 clean 后，才允许正常启动。

## 当前验证结论

- 单元测试：`pixi run python -m pytest -q tests/test_model_security_bypass_and_metrics.py tests/test_model_security_runtime.py` 通过。
- 全量编译：`pixi run python -m compileall -q src tests` 通过。
- 接口验证：`/api/model-security/trust` 可返回历史净化记录的原始源模型与 ASR 对比。
- 页面验证：安全中心和主监控页都能看到测试绕过入口，主监控页能显示该入口的研发警告文案。

## 后续建议

- 未知模型如果没有可用验证集，不能凭空计算真实原始 ASR，只能显示 `unverifiable` 或依赖已知包内证据。
- 后续可以把 ASR 对比扩展为独立报告页，展示攻击家族、样本数、成功数、代表图片和报告路径。
- 测试绕过入口在 B 模块验收稳定后，应改为隐藏开发选项或由配置显式启用。
