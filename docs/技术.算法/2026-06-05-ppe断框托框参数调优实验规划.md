# PPE 断框/托框参数调优实验规划

## 实验目标

本实验用于形成项目交底书中的 PPE 检测框稳定性调优资料。目标是在不修改模型权重、不改变 person/head/helmet 类别语义、不破坏 A3b/Module A 策略的前提下，找到一组更均衡的运行时与后处理参数，使 head/helmet/person 的画面框在多人、远距离和短时遮挡场景中更稳定，同时避免旧框滞留、托框、串框和 2 人共框。

一句话目标：减少 head/helmet 偶发断框，但不能把断框问题替换成托框或错人绑定。

## 当前问题边界

断框不是单一模型问题，也不是单一参数问题。当前链路至少包含三层：

1. 原始检测层：由模型输出、`confidence`、`candidate_confidence` 和 NMS IoU 决定某帧是否有候选框。
2. PPE 后处理/显示跟踪层：由 `PPEDisplayTracker` 将原始 person/head/helmet 候选转换为稳定显示轨迹。
3. Web 预览 overlay 层：由后端预览时钟、latest-only 检测队列和 overlay 桥接窗口决定框是否继续显示到画面。

因此实验必须先定位断在哪一层，再调整参数。禁止直接大幅降低模型置信度或大幅拉长保持窗口。

## 代码链路依据

- 原始检测进入：`model/src/defense/module_a/backends/detector_backend.py`
- PPE 汇总与抑误报：`model/src/defense/module_a/ppe_postprocess.py`
- PPE 业务状态：`model/src/defense/runtime/ppe_business.py`
- PPE 显示跟踪：`model/src/defense/module_a/postprocess/ppe_tracking.py`

## 待调参数族

### 原始检测层

- `inference.confidence`
- `inference.candidate_confidence`
- NMS IoU

使用原则：最后再动。降低原始置信度会增加小目标召回，但也最容易引入 head/helmet 误检。

### PPE 显示跟踪层

- `hold_frames`
- `small_hold_frames`
- `iou_match_threshold`
- `max_missed_ms`
- `smooth_alpha`
- `weak_promotion_hits`
- `weak_head_min_avg_confidence`
- `weak_helmet_min_avg_confidence`
- `weak_helmet_isolated_min_avg_confidence`
- `weak_edge_promotion_hits`
- `weak_edge_min_avg_confidence`

使用原则：优先小步调整。远距离小目标可适度增加 `small_hold_frames` 和弱证据晋升条件，但多人重叠场景必须回归验证，避免串框。

### 预览 overlay 层

- `max_render_misses`
- `overlay_match_window_ms`
- `overlay_hold_ms`
- `overlay_interpolate_ms`
- `overlay_max_age_ms`

使用原则：这是断框与托框的核心平衡层。当前文件实时预览会主动缩短桥接窗口，目的是避免旧框拖尾。调大可减少断框，但必须检查旧框滞留。

## 实验数据集与场景切片

至少覆盖以下片段：

- 当前固定镜头室外视频：多人、远距离、强光背景。
- 多人重叠/交叉行走片段：验证 2 人共框、错人绑定和 track id 切换。
- 远距离小目标片段：验证 head/helmet 召回和弱证据晋升。
- 前景大人经过片段：验证小目标不会被大 person 遮挡后长期托框。
- person 绘制开启与关闭两种显示状态：验证 `person` 只影响绘制，不影响 PPE 业务判断。

## 诊断采集

每个候选参数组至少导出逐帧诊断表，字段建议如下：

- `frame_idx`
- `video_time_s`
- `raw_boxes_count`
- `ppe_raw_class_counts`
- `ppe_class_counts`
- `ppe_person_count`
- `ppe_head_count`
- `ppe_helmet_count`
- `ppe_missing_helmet_count`
- `ppe_tracks`
- 每个 track 的 `track_id`、`label`、`stable_label`、`confidence`、`box`、`source`、`misses`、`is_small`、`hold_eligible`
- `display_options`
- `overlay_display_source`
- `processing_ms`
- `detector_inference_ms`
- `preview_fps`

诊断目标是区分四类原因：

- 原始模型无框；
- 原始有框但 PPE 后处理抑制；
- PPE 有轨迹但 overlay 不保持；
- overlay 有轨迹但显示配置隐藏。

## 评价指标

### 稳定性指标

- 断框事件数：同一目标在短时间内框消失又出现。
- 平均断框长度：连续缺失帧数或毫秒数。
- 最长断框长度：最坏情况。
- 标签翻转次数：head/helmet 互相跳变。
- track id 切换次数：同一目标频繁换 id。

### 负面副作用指标

- 托框次数：目标离开或位置明显变化后旧框仍停留。
- 旧框滞留时长：旧框保留超过真实目标的毫秒数。
- 多人串框次数：框从一个人跳到另一个人。
- 2 人共框次数：多人重叠时错误合并成一个 person/head/helmet 显示。
- 误检增加量：弱证据晋升后新增的错误 head/helmet。

### 性能指标

- `processing_ms`
- `detector_inference_ms`
- `preview_fps`
- `dropped_detection_frames`
- `stale_overlay_dropped`

任何参数组如果显著增加处理耗时或导致预览不同步，应降级或淘汰。

## 参数组设计

### A0 基线组

完全使用当前参数，作为对照组。必须保留基线结果视频、逐帧诊断和 3 秒抽帧记录。

### A1 保守稳定组

只微调 overlay 和 `max_render_misses`，不动原始检测置信度。目标是减少 1-2 帧偶发断框，同时尽量不增加托框。

### A2 小目标增强组

在 A1 基础上适度增加 `small_hold_frames`，并微调弱 head/helmet 晋升条件。目标是改善远距离人员只有 person、head/helmet 断续的问题。

### A3 多人重叠防串组

在 A1/A2 中选择较优配置，再收紧匹配或保持窗口，专门验证多人重叠时不共框、不串框。

### A4 高召回候选组

仅作为对照，轻微降低 `candidate_confidence` 或弱证据阈值。该组风险最高，只用于判断模型原始召回瓶颈，不作为默认部署首选。

## 实验流程

1. 记录当前配置与 Git 状态。
2. 启动 Web 服务：`pixi run monitor` 或 `pixi run monitor-open-external`。
3. 通过 Web/API 跑同一视频，保存基线检测结果。
4. 导出逐帧诊断 JSONL/CSV。
5. 按 A1-A4 参数组小网格运行。
6. 为每组生成检测结果视频。
7. 对每组随机抽连续 3 秒完整帧，逐帧检查断框、托框、串框、标签翻转和遮挡异常。
8. 汇总指标表，淘汰副作用明显的参数组。
9. 形成推荐参数、适用场景和禁止使用条件。

## 验收标准

推荐参数组必须同时满足：

- 断框事件数低于基线；
- 最长断框长度低于基线或不恶化；
- 托框、旧框滞留不高于基线；
- 多人重叠场景不出现新增 2 人共框；
- head/helmet 标签翻转不高于基线；
- `processing_ms` 与 `preview_fps` 无明显退化；
- person 隐藏显示开关不影响状态计数和 PPE 业务判断。

如果没有参数组同时满足上述条件，则结论应写为“当前断框主要来自原始模型证据不足或场景难度，运行时参数只能有限缓解”，不得强行推荐。

## 视觉验收要求

凡涉及检测效果、检测框显示、overlay 或流畅度的参数变更，提交前必须：

- 用当前项目实际跑目标视频；
- 生成检测结果视频；
- 从结果视频随机抽取连续 3 秒；
- 导出该 3 秒内完整帧；
- 逐帧肉眼检查；
- 记录视频路径、抽帧区间、抽帧图片路径和检查结论。

发现拖框、断框、旧框滞留、明显不流畅、状态不同步、误导性显示时，不允许提交。

## 预计耗时

- 快速基线与诊断方案：30-60 分钟。
- 快速候选参数：1-2 小时。
- 中等可信实验：3-5 小时。
- 多场景完整调优：约 1 天。

建议第一轮采用 3-5 小时中等实验，先得到可解释的候选参数和交底资料，再决定是否扩大到完整多场景调优。

## 交底书结论模板

本项目 PPE 检测框稳定性不是单纯模型阈值问题，而是原始检测、PPE 后处理时序、Web 预览 overlay 三层共同决定。实验采用基线组、保守稳定组、小目标增强组、多人重叠防串组和高召回候选组进行对比，评价断框、托框、串框、标签翻转和性能影响。推荐参数必须在减少断框的同时不增加旧框滞留和多人错绑；若所有参数组均无法满足，则应将问题归类为模型证据不足或场景难度，需要进入模型数据/训练侧处理。
