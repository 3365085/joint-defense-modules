# 项目介绍、Module A 实现与数据风险

日期：2026-07-18
范围：主项目 `model/` 的生产默认链路。
口径：代码、配置、artifact 元数据和权威验收 manifest。历史 `rebuilt_demo/` 不作生产依据。

## 1. 项目做什么

系统保护 PPE 视频检测链路：

- Module A 看运行中的视频。找物理扰动、翻拍、遮挡、强光、模糊、可见性退化、对抗补丁。
- Module B 看运行前的模型。做哈希准入、投毒扫描、净化、白名单和导出。
- YOLO 做 `helmet/head/person` 目标检测。Module A 不替代 YOLO；它使用画面、YOLO ROI 和时序证据判断检测链路是否受攻击。
- Web 只展示最终状态。真实决策在 FastAPI、MonitorEngine、VideoDefensePipeline 和 Module A detector 中完成。

## 2. 生产数据流

1. 文件源优先走 `PyNvVideoCodec`/NVDEC。失败可回退 CPU；原因必须可见。
2. GPU 解码帧保留可复用 CUDA RGBP tensor。TensorRT/Ultralytics 预处理尽量留在 GPU。
3. YOLO 输出统一成 bbox、class、confidence 和耗时契约。
4. PPE 后处理维护人员、安全帽、头部和显示连续性。
5. Module A 同时读取当前帧、上一帧、场景基线和 YOLO ROI。
6. A1/A2/A3/A3b/A4/Blind 产出分数、候选、抑制原因和 bbox。
7. 联合决策做目标关系检查、场景抑制、救援规则、N-of-M 确认和短时保持。
8. preview 与 detection 解耦。检测队列使用 `latest_only`；落后时丢旧检测帧，不阻塞预览。
9. evidence 异步写盘。运行状态暴露 backend、artifact hash、native、flow、fallback 和队列健康。

## 3. Module A 具体技术

### 3.1 A1：单帧纹理异常

输入：当前灰度帧的 LBP、YOLO ROI、场景 LBP 基线。

计算：

- 全局 LBP 直方图和场景基线距离。
- `8 x 8` 网格局部最大偏移、局部均值、空间集中度。
- 人/头/安全帽 ROI 内纹理偏移和 `4 x 4` 子块极值。
- bbox 与目标的 IoU、邻近度、面积关系。
- 曝光变化、冷启动和静止背景抑制。

输出：`a1_feature_score`、`A1_LBP_SINGLE` 候选、局部 bbox、目标关系。Rust 算子 `a1_lbp_features` 可把大量直方图循环压成一次原生调用；不可用时回退 NumPy/OpenCV。

### 3.2 A2：时序纹理异常

输入：当前/上一帧 LBP、ROI、光流、曝光上下文。

计算：

- 全局、网格、ROI 和 ROI 外环的 LBP 变化。
- 局部突发与上下文对比。
- 用光流判断纹理变化能否被正常运动解释。
- 区分局部异常、全局曝光变化和闪烁。

输出：`a2_feature_score`、`A2_LBP_TEMPORAL`、`flash_like` 和未被运动解释的纹理变化。Rust 算子 `a2_change_features` 提供原生路径。

### 3.3 A3：异常运动与光流残差

光流后端顺序：

1. RAFT-small TensorRT，`256 x 256`，独立 CUDA stream。
2. GPU Lucas-Kanade，无模型权重。
3. OpenCV DIS CPU 回退。

每次回退记录 `flow_backend`、`flow_effective_device`、`flow_fallback_reason`。

计算：

- 去除全局平移后的 residual flow。
- 网格与 ROI 残差、ROI 外环残差、内外运动差。
- 异常像素比例、形状集中度、背景一致性。
- 异常流是否落在 YOLO 目标内。目标正常运动能解释时压低分数。
- 高残差短时保持，减少一帧命中后立刻断开。

输出：`a3_feature_score`、`A3_FLOW_ARTIFACT`、局部 bbox、flow 后端状态。

### 3.4 A3b：静态媒体、屏幕和纸张翻拍

A3b 慢于主路径。生产实现放后台 worker：主线程拿最近一次有效结果，不等待。worker 有 generation、timeout、result lease、进程级数量上限和错误状态。

主要步骤：

- L0：边缘、矩形、面积、边界和显示框候选。
- L1：候选 IoU 跟踪、命中/丢失计数、bbox 抖动和寿命。
- L2：矩形/边缘平面启发式、flow residual、候选内外运动差。
- YOLO 上下文：候选与 person/head/helmet 的 IoU、邻近度和面积比。
- 类型评分：`screen_replay`、`paper_photo`、`static_image_spoof`、`flat_media_candidate`。
- policy：抑制墙面、窗框、字幕边框、相机运动、物理贴纸和普通背景平面。

默认 rebuilt 路径不执行 KLT/RANSAC Homography。结果中的
`homography_inlier_ratio` 是兼容字段名，当前值来自矩形、边缘和 flow-gap
启发式。Moiré 当前只作诊断，不直接触发。A3b 结果也会帮助区分“真实媒体翻拍”和“目标附着物理补丁”。

### 3.5 A4：96 维 XGBoost 救援分类器

生产 artifact：`model/runtime/artifacts/module_a/a4/a4_classifier.pkl`。

当前 schema：`rebuilt-a4-96-v4`。

- A1 基础特征：5 维。
- A2 基础特征：5 维。
- A3 基础特征：6 维。
- 当前帧 patch 统计：40 维。覆盖 saturation、value、chroma、gradient、laplacian、亮暗/高色度/边缘比例的全局和 tile 极值。
- 相对前 12 个采集帧中值的 patch delta：40 维。

总计 `96` 维。模型类型 `xgboost.sklearn.XGBClassifier`。角色是 `adv_patch_rescue`，不是所有攻击的唯一裁判。

运行语义：

- A4 同步使用当前帧 A1/A2/A3 和 patch 特征。
- A4 不读取异步 A3b cache，避免时间错位和 schema 漂移。
- classifier 成功时，`p_adv = max(rule_p_adv, classifier_p_adv)`。
- classifier 缺失、哈希不符、schema 不符或预测失败时，禁用该 classifier，继续走规则融合；错误进入状态。
- 当前 metadata 阈值 `0.94`，告警窗口 `8` 帧，至少 `5` 次命中。

内部 heldout：36 clips、18 scenes、2778 rows；视频级攻击召回约 `0.8889`，AUC 约 `0.9621`，clean FP clips 为 `0`。这些是合成数据内部门禁，不是现实生产保证。

### 3.6 Blind：致盲和信号退化

Blind 不只看绝对模糊值。它维护当前场景自身基线：

- Laplacian 方差：清晰度。
- 灰度标准差：对比度。
- YOLO confidence 总量：目标可见强度。
- 过曝比例、短时清晰度骤降、目标骤失和全局运动。

相对基线突然退化并影响目标，才形成主要证据。长期低清晰度场景不会只因绝对值低就报警。Rust `blinding_laplacian_var` 可加速单帧清晰度统计。

### 3.7 联合决策与持续性

联合决策不是单阈值：

- A4 规则/XGBoost、Blind、A3b 分路产生候选。
- 检查目标关系、多证据协同、曝光、背景运动、静止背景和媒体类型。
- 按实际处理 FPS 生成约半秒窗口。
- 使用 N-of-M 命中确认；高误报场景要求更多命中。
- 已确认事件短时 hold；新源、seek、时间倒退或生命周期重启时清状态。
- Web overlay 按 source frame/timestamp 匹配，不能把旧框贴到新帧。

## 4. 性能技术

- **解码**：文件优先 NVDEC；派生缓存和 fallback 原因可观察。
- **推理**：权威 YOLO TensorRT FP16；ONNX/PyTorch 留作兼容路径。
- **GPU 预处理**：YOLO 路径在 CUDA RGBP tensor 上做 resize、归一化和 dtype 转换；Module A 仍消费 host BGR 帧，不宣称全链零拷贝。
- **光流**：RAFT TensorRT 优先，GPU LK 次级，DIS CPU 最后。
- **原生算子**：Rust + PyO3 + NumPy，release 开启 LTO；覆盖 A1、A2、A3 网格、A3b 批统计和 Blind Laplacian。
- **可信加载**：native 校验 API、crate 版本、capabilities、二进制 hash、源码 manifest hash和构建时源码证明。
- **并发**：preview/detection 分离，`latest_only` 控背压；A3b 和 evidence 后台运行。
- **可观察性**：GPU/CPU、artifact、native、flow、decoder、fallback、队列、耗时都进入 runtime status。

## 5. 数据少：主要检测风险

### 5.1 权威正例极少

权威 Web 验收 manifest 共 `36` 段：

- 正常：`30` 段。
- 物理攻击：`5` 段，adv patch、glare、motion blur、occlusion、visibility degradation 各 `1` 段。
- A3b：`1` 段。

每类只有一段正例。无法估计同类攻击在不同设备、人物、距离、角度、光照和背景下的召回分布。

### 5.2 帧多不等于样本多

相邻视频帧高度相关。一个 30 秒视频可产生上百训练行，但独立场景仍只有一个。把帧级 AUC 当成场景级泛化，会高估能力。

### 5.3 合成捷径

A4 训练 provenance 明确使用 `a4-synthetic-carrier-v2`。合成贴纸、轨迹、颜色、边缘、载体和编码器可能形成捷径。模型可能识别生成器风格，不是现实物理攻击。

### 5.4 验收集参与开发

权威视频已用于调参、回归和人工观察。它们适合当前交付验收，不再是严格独立终测集。继续用同一批视频改阈值，会进一步产生评测泄漏。

### 5.5 正常样本覆盖不足

当前正常集不能覆盖：

- 夜间、雨雪、雾、逆光、镜头脏污。
- 手机防抖、云台、快速摇摄、自动曝光和自动对焦。
- H.264/H.265 不同码率、B 帧、VFR、网络丢包和 RTSP 卡顿。
- 多人交叉、远距离小目标、强反光 PPE、广告屏、窗框、门框和字幕。

新域可能同时提高误报和漏报。

### 5.6 当前能说什么

可以说：

- 当前代码契约、artifact 绑定、回退可见性和指定样本链路已验证。
- 当前指定视频上达到记录中的效果。

不能说：

- 已证明跨摄像头泛化。
- 已得到真实生产误报率。
- 对所有物理攻击稳定召回。
- 可替代人工复核或安全联锁。

## 6. 补数据顺序

1. 每种攻击至少采集多个独立 `场景 x 设备 x 相机 x 光照 x 距离` 组合。
2. A3b 单独补屏幕、手机、平板、纸张、不同刷新率、不同拍摄角度和困难负样本。
3. 划分按 scene/device/video，不按 frame。
4. 冻结一套从未参与调参的最终测试集；只在发布候选上运行一次。
5. 正常流做小时级测试，统计每小时误报和每视频误报，不只统计帧级 FP。
6. 每类报告视频级 recall、首次触发延迟、持续性、误报率和置信区间。
7. 跨 H.264/H.265、分辨率、FPS、VFR、摄像头和 RTSP 做系统门禁。

- `model/src/defense/module_a/rebuilt/detector.py`：生产 A1/A2/A3/A3b/A4/Blind 与联合决策。

- `model/src/defense/module_a/rebuilt/a4_artifact.py`：A4 metadata、hash、schema 和质量门禁。
- `model/src/defense/module_a/native_bridge.py`：Rust 二进制可信加载与 fallback。
- `model/native/module_a_native/src/lib.rs`：原生算子。

- `model/src/defense/pipelines/video_defense_pipeline.py`：YOLO、PPE、Module A、叠框和时序接线。

- `model/configs/module_a_runtime.yaml`：生产有效配置。
- `model/configs/acceptance/module_a_authoritative_manifest_v1.json`：权威模型与视频口径。
