# 手工困难负样本标注包修正记录

## 背景

固定镜头室外视频中出现“手被模型识别为 head”的模型本体误检。该问题不能继续使用旧模型输出或 reference 检测框自动生成训练标签，必须通过人工标注困难帧来补充训练数据。

## 本次问题

手工标注 Web 界面初版只应展示 331 张待标注帧，但用户发现第 2 至第 13 张画面重复。检查待标注图片文件哈希后确认，旧数据包中的多张 JPEG 内容完全一致，不是浏览器缓存或列表刷新问题。

## 当前判断

旧打包脚本使用 OpenCV `CAP_PROP_POS_FRAMES` 对 MP4 做逐帧随机 seek。该方式在当前视频和路径环境下返回了重复帧，导致数据包污染。已将抽帧方式改为顺序读取视频，到达目标帧号时再落盘，重建的新包前 15 张哈希均不同。

旧包中已有 3 个手工标签，但对应的新旧图片哈希均不一致，因此不能自动迁移到新包训练，以免把框标到错误画面上。

## 影响范围

- 影响：`D:/defense_purification_data/manual_hardneg_fixed_outdoor_20260611`
- 修正包：`D:/defense_purification_data/manual_hardneg_fixed_outdoor_20260612_seq`
- 标注服务默认数据包已切到修正包。

## 后续建议

继续只使用人工标注或人工审核后的标签作为本场景困难样本。训练前需要再次检查数据包重复帧、空标签比例、train/holdout 划分，并保留 holdout 帧只做验收，不参与训练。
