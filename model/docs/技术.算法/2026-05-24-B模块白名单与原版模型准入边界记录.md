# B模块白名单与原版模型准入边界记录

## 背景

用户确认新的产品口径：模型进入 A 模块生产检测前，必须先由 B 模块对原版格式模型执行完整检测；如果用户需要 TensorRT engine、ONNX 等加速格式，应在原版模型检测达标后再由系统生成或绑定加速 artifact，并计算哈希写入白名单。白名单不能由用户手工新增或修改，用户只允许删除已有白名单记录。

## 当前判断

1. `.pt` / `.pth` 是 B 模块 full scan 和后续净化/重训的可信输入。它保留 PyTorch 权重、类别语义和可训练/可分析结构，适合作为安全准入和净化依据。
2. `.engine` 是 TensorRT 编译后的运行 artifact，主要用于推理加速，不保留可直接训练和净化所需的原始计算图语义。它可以作为运行模型被哈希绑定到已验证 PT，但不应作为 B 模块神经元级扫描、净化或重训对象。
3. ONNX 同样更偏运行/交换格式，不作为当前净化主对象；若作为运行 artifact 使用，也应绑定同源、已通过 full scan 的 PT。
4. 白名单命中必须依赖模型文件 hash、源 PT hash、类别表 hash、PPE 语义 hash 和 scanner_version，不能只靠路径。

## 代码链路依据

- `defense.model_security.service.ModelSecurityService.admission_status`：负责准入状态判断、source PT 解析、registry 命中检查。
- `defense.model_security.service.ModelSecurityService.scan`：full scan clean 后自动调用 `ModelTrustRegistry.mark_trusted` 写入白名单。
- `defense.model_security.scanner.full_scan`：要求 `source_model_path` 为 `.pt` / `.pth`，并用源 PT 配置执行外部验证。
- `defense.model_security.registry.ModelTrustRegistry`：白名单持久化、查询、删除和清空。
- `defense.web.fastapi_app.create_app`：`/api/start` 在调用 `MonitorEngine.start` 前先执行 B 准入；`/api/model-security/trust/delete` 和 `/api/model-security/trust/clear` 只提供删除能力。

## 影响范围

- 默认 engine 可以继续作为 A 模块运行 artifact，但 B full scan 的验证输入应切到同源 `best.pt`。
- 自定义加速 artifact 若无法解析同源 PT，应进入 `unverifiable`，不允许启动生产检测。
- clean full scan 自动写入白名单；`review`、`suspicious`、`unverifiable` 不写入。
- 用户界面展示白名单状态，并允许删除当前模型或清空白名单；不提供手工新增白名单入口。

## 本次发现的问题

自定义 engine 的 source PT 推断不能复用默认配置里的 `inference.artifacts.pytorch`。否则当用户选择 YOLOv8 engine 时，可能错误绑定默认 YOLOv5 `best.pt`。修复后，`ModelSecurityService._source_pt_candidates` 在 `runtime.custom_model.enabled=true` 时只使用自定义模型显式 source PT 或同目录 `best.pt` / 同名 `.pt`，不再回退到默认 artifact。

## 结论

白名单应是系统根据 full scan clean 结果生成的“显式白名单”，不是用户可编辑配置。加速模型的可信性来自“同源 PT 已通过 B 模块 full scan + 加速 artifact hash 绑定”，而不是直接扫描或净化 `.engine`。

## 后续建议

1. 增加“生成加速模型”的受控任务入口：只接受已通过 full scan 的 PT，导出完成后自动计算 runtime artifact hash 并更新白名单绑定。
2. 对 YOLOv5 / Ultralytics PT 分别维护适配测试，避免类别表、backend、source PT 推断错误导致误判为不支持。
3. 将 B 模块 external hard suite 的目标类、验证集和 ASR 判定继续参数化，避免 PPE 小目标类别与通用后门检测口径混用。
