# 每帧检测、隔帧复用与多帧报警链路记录

## 背景

用户询问：PPE 每帧检测和 A1-A4 模块是否冲突；当前 A1-A4 是否也是每帧检测；哪些逻辑是隔帧检测；哪些逻辑依靠多帧报警。

## 代码链路依据

- `src/defense/runtime/frame_processor.py:FrameProcessor.process()` 是每个处理帧的入口。
- `src/defense/pipelines/video_defense_pipeline.py:VideoDefensePipeline._run_detection()` 先运行 YOLO 检测/复用，再把 ROI 传给 `ModuleADetector.process()`。
- `src/defense/module_a/process_pipeline.py:process_module_a()` 负责 A1、A2、A3、A3b、A4。
- `src/defense/module_a/detector.py:_should_run_light_flow()` 控制 light-flow 隔帧/候选触发运行。
- `src/defense/module_a/detector.py:_should_run_static_image()` 控制 static-image ROI pass 隔帧/候选触发运行。
- `src/defense/module_a/alert_state.py:AlertState.update()` 是 A 模块多帧确认状态机。
- `src/defense/runtime/ppe_state.py:SafetyHelmetState.update()` 是 PPE 多帧确认状态机。
- `src/defense/runtime/a3b_soft_trigger.py:A3BSoftTriggerState.update()` 是 A3b 的窗口/连续命中软触发状态机。

## 当前结论

1. PPE 与 A1-A4 不属于同一告警链路，但共用 YOLO 检测结果。
   YOLO 检测框一方面给 PPE 后处理用，另一方面经 `DetectionROIProvider` 生成 ROI 供 A1-A4 的局部特征使用。二者不会直接互相覆盖告警，但会在性能预算、ROI 质量和显示状态上互相影响。

2. A1-A4 是“每个处理帧都进入流程”，但不是所有重算子每帧完整运行。
   A1 过曝、A2 LBP/时序纹理、A3 motion/blur/track、A4 fusion 都在每个处理帧进入 `process_module_a()`。其中 light-flow 和 static-image ROI pass 有隔帧/候选触发门控。

3. YOLO 不是绝对每帧推理。
   `VideoDefensePipeline._maybe_reuse_detections()` 会在画面变化很小时复用上一帧检测结果。配置项为 `temporal_detector_reuse_threshold` 和 `temporal_detector_reuse_max_gap`。

4. 实时文件/视频输入不是处理每一个原始视频帧。
   运行配置中 `process_fps_cap` 和 `detector_process_fps_cap` 默认为 15，预览为 25fps；后端使用 latest-only backpressure。也就是说高帧率视频会预览更多帧，但检测处理按预算抽样最新帧。

5. 多帧报警逻辑有三套：
   - A 模块 p_adv：`AlertState.update()`，配置 `alert_window`、`alert_trigger_count`。
   - PPE：`SafetyHelmetState.update()`，默认 6 帧窗口、3 次候选确认、12 帧 hold。
   - A3b：`A3BSoftTriggerState.update()`，配置 `window_size`、`min_window_hits`、`min_consecutive_hits`、`trigger_hold_frames`。

## 对方案 3 的影响

`weakHead` 时序升级应该接在 PPE 后处理/显示跟踪链路内，不应该插入 A1-A4 的 p_adv 告警状态机。这样可以避免 PPE 安全帽语义和物理扰动检测语义混在一起。

方案 3 需要注意：

- 不要增加 YOLO 主路径额外全帧推理。
- 不要改变 A1-A4 的 p_adv 状态机。
- 只对 PPE 的 `head_count` / `missing_helmet_count` / `candidate` 做受限升级。
- 升级后的 PPE 候选仍应经过 `SafetyHelmetState` 多帧确认，而不是单帧直接 confirmed。
