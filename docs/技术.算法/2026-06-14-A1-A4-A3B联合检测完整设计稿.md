# A1-A4 与 A3B 联合检测完整设计稿

日期：2026-06-14

## 1. 设计目标

本设计用于重建模块 A 的物理扰动检测链路，使其满足两个核心目标：

1. 正常视频不应被持续报警，尤其是室内外光照变化、固定室外镜头、行人正常运动、背景边框和广告牌等正常场景。
2. 攻击视频应在攻击出现后触发，且每个攻击视频尽量由一个主导模块给出高分解释，避免所有模块同时虚高。

该设计不是重新命名 A1-A4，也不是把已有工程字段硬塞进基础文档。基础 A1-A4 必须严格遵循 `doc/前置文档/基础文档.txt`：

- A1：LBP 单帧纹理异常，输出 `ΔH`。
- A2：LBP 时序纹理不一致，输出 `Change_t`。
- A3：光流伪影检测，输出 `f_flow`。
- A4：融合分类，输入 `ΔH + Change_t + f_flow`，输出 `p_adv`。

A3B 是模块 A 的工程扩展分支，专门检测假视频、屏幕翻拍、纸张照片和静态介质攻击。A3B 不属于基础 A1-A4 主链路，不改变 A4 的 `p_adv` 定义，而是独立输出 `p_media`，交给后处理/联合决策层统一处理。

## 2. 名词边界

### 2.1 A1-A4 的稳定定义

| 模块 | 基础定义 | 主输出 | 是否直接实时告警 |
| --- | --- | --- | --- |
| A1 | LBP 单帧纹理异常 | `delta_h` / `a1_feature_score` | 否 |
| A2 | LBP 时序纹理变化 | `change_t` / `a2_feature_score` | 否 |
| A3 | 光流伪影检测 | `f_flow` / `a3_feature_score` | 否 |
| A4 | 融合分类 | `p_adv` | 只产生单帧候选 |
| A3B | 假视频/静态介质工程扩展 | `p_media` | 只产生介质候选 |
| 后处理/联合决策层 | 误报抑制、短窗确认、解释 | `alert_confirmed` | 是 |

### 2.2 `p_adv` 的语义

`p_adv >= θ_adv` 表示“单帧候选报警/标记”。它保留基础文档中“超过阈值则报警”的单帧标记逻辑，但最终实时声光/弹窗告警必须经过后处理/联合决策层的短窗确认。

因此工程上必须拆分以下概念：

- `adv_single_frame_candidate`：A4 单帧候选，来自 `p_adv >= θ_adv` 且通过 adv 门控。
- `media_single_frame_candidate`：A3B 单帧候选，来自 `p_media_policy >= θ_media` 且通过 media 门控。
- `single_frame_candidate`：总单帧候选，等于 `adv_single_frame_candidate or media_single_frame_candidate`。
- `alert_confirmed`：实时确认告警，来自后处理/联合决策层。

实现时必须把 A4 候选和 A3B 候选拆开记录，不能只保留一个含混的 `single_frame_candidate`。`single_frame_candidate` 只能作为总候选字段，便于 UI 和统计汇总。

### 2.3 后处理/联合决策层的边界

后处理/联合决策层不是 A4，也不是新的攻击检测模块。它只做三件事：

1. 场景门控：抑制明显正常场景误报。
2. 短窗确认：把单帧候选转成实时确认告警。
3. 结果解释：解释 A1/A2/A3/A3B 哪一路证据主导。

它不能改变 A1/A2/A3/A4 的原始定义，不能把 `p_media` 混入 `p_adv`，也不能把显示放大分数当作真实检测概率。

## 3. 总体架构

基础 A1-A4 是三路并行特征提取后进入 A4，而不是 A1 -> A2 -> A3 串行。

```text
视频帧 Ft 及相邻帧 Ft-1, Ft + YOLO 目标框
   │
   ├─ A1：LBP 单帧纹理异常
   │      输入：Ft、目标 ROI、背景 ROI
   │      输出：ΔH、局部纹理异常、a1_feature_score
   │
   ├─ A2：LBP 时序纹理不一致
   │      输入：LBP(Ft-1)、LBP(Ft)、目标 ROI
   │      输出：Change_t、局部时序变化、a2_feature_score
   │
   ├─ A3：光流伪影检测
   │      输入：Ft-1、Ft、目标 ROI
   │      输出：f_flow、局部光流异常、a3_feature_score
   │
   └─ A3B：假视频/静态介质工程扩展分支
          输入：Ft-1、Ft、YOLO 目标框
          输出：p_media、p_media_type、p_media_bbox、target_related

        A1/A2/A3
           ↓
      A4：融合分类
           输入：ΔH + Change_t + f_flow
           输出：p_adv

        A4 p_adv + A3B p_media_policy + 场景上下文
           ↓
      后处理/联合决策层
           ├─ 场景门控 / 误报抑制
           ├─ 短窗确认
           └─ 结果解释
           ↓
      single_frame_candidate / alert_confirmed / dominant_input / reason
```

## 4. A1 设计：LBP 单帧纹理异常

### 4.1 职责

A1 只回答一个问题：当前单帧图像中，目标附近或局部区域的纹理分布是否明显偏离正常背景/目标纹理。

它适合捕捉：

- 对抗衣物、贴纸、纹理块带来的局部纹理异常。
- 局部过曝、遮挡边界、异常图案导致的 LBP 分布异常。
- 目标 ROI 内部纹理突然不符合周边统计的情况。

它不适合单独判断：

- 正常光照变化。
- 正常衣服花纹。
- 摄像头运动导致的整体变化。
- 屏幕/纸张介质语义。

### 4.2 输入

- 当前帧灰度图或亮度图 `gray_t`。
- 当前帧 LBP 图 `lbp_t`。
- YOLO 目标框 `target_rois`。
- 可选背景网格 ROI `background_rois`。

### 4.3 核心特征

- `delta_h_global`：全图 LBP 直方图异常强度。
- `delta_h_roi_max`：目标 ROI 内最大纹理异常。
- `delta_h_local_max`：局部窗口最大纹理异常。
- `delta_h_target_contrast`：目标 ROI 与周边背景的纹理差异。
- `delta_h_spatial_concentration`：异常是否集中在目标附近，而不是全图均匀变化。

### 4.4 打分原则

A1 的 `a1_feature_score` 不应简单等于全局 `delta_h`。更合理的形式：

```text
a1_feature_score =
  target_local_strength
  * spatial_concentration
  * target_relation_weight
  * normal_scene_penalty
```

其中：

- `target_local_strength` 高，表示目标局部纹理异常强。
- `spatial_concentration` 高，表示异常集中，不是全局光照。
- `target_relation_weight` 高，表示异常与目标框或目标附近有关。
- `normal_scene_penalty` 用于降低整帧亮度/曝光变化造成的误报。

### 4.5 输出契约

- `delta_h`
- `delta_h_global`
- `delta_h_local_max`
- `delta_h_roi_max`
- `delta_h_target_contrast`
- `a1_feature_score`
- `a1_candidate`
- `a1_reason`

### 4.6 抑制规则

A1 不直接实时报警。以下情况应降权：

- 全图 LBP 同步变化，但局部集中度低。
- 室内到室外导致整体亮度和纹理直方图变化。
- 目标 ROI 与背景 ROI 同时变化，且变化幅度接近。
- 单帧尖峰后立即恢复。

## 5. A2 设计：LBP 时序纹理不一致

### 5.1 职责

A2 只回答一个问题：相邻帧的 LBP 特征图是否发生了不符合正常运动的变化。

它适合捕捉：

- 闪烁光照导致的 LBP 特征图快速变化。
- 对抗贴片移动导致的局部纹理时序突变。
- 遮挡/反光/局部干扰导致的局部纹理不连续。

需要注意：闪烁光照不是 A2 的专属检测目标，只是可能引起相邻帧 LBP 特征图变化，表现为 `Change_t` 升高。

### 5.2 输入

- `lbp_{t-1}`
- `lbp_t`
- 目标 ROI。
- 目标运动估计或光流辅助信息。

### 5.3 核心特征

- `change_t_global`：全图 LBP 时序变化。
- `change_t_roi_max`：目标 ROI 内最大时序变化。
- `change_t_local_max`：局部最大时序变化。
- `change_t_without_motion_target`：纹理变化强，但缺少对应正常运动目标。
- `change_t_motion_aligned`：纹理变化是否能被目标运动解释。
- `change_t_burst`：短时间内是否连续高变化。

### 5.4 打分原则

A2 的关键不是“变化大就报警”，而是“变化大且不像正常运动”。

```text
a2_feature_score =
  temporal_texture_change
  * local_burst_weight
  * no_corresponding_motion_weight
  * target_relation_weight
  * exposure_penalty
```

其中：

- 正常行人运动会带来 LBP 变化，但应被运动对齐规则降权。
- 全局曝光或摄像头自动增益会带来全局 `Change_t`，但局部集中度低，应降权。
- 与目标框重叠或邻近的局部异常，比远处背景异常更重要。

### 5.5 输出契约

- `change_t`
- `change_t_global`
- `change_t_local_max`
- `change_t_roi_max`
- `change_t_without_motion_target`
- `change_t_motion_aligned`
- `a2_feature_score`
- `a2_candidate`
- `a2_reason`

## 6. A3 设计：光流伪影检测

### 6.1 职责

A3 只回答一个问题：相邻帧光流场中是否存在不符合正常目标运动的局部伪影。

它适合捕捉：

- 动态遮挡。
- 移动对抗贴片。
- 局部扰动区域与人体整体运动不一致。
- 目标附近出现异常光流边界或残差。

它不应把所有相机运动、行人正常运动、背景大面积运动都当作攻击。

### 6.2 输入

- `gray_{t-1}`
- `gray_t`
- 目标 ROI。
- 可选轻量光流结果、帧差结果、KLT/稀疏点跟踪结果。

### 6.3 核心特征

- `flow_local_anomaly_ratio`：局部光流异常占比。
- `flow_max_magnitude_norm`：局部最大运动强度归一化。
- `flow_residual`：局部运动与周边/目标整体运动的残差。
- `flow_shape_score`：异常区域形状是否像扰动区域，而非整块相机运动。
- `flow_target_relation`：异常区域与目标框的关系。
- `flow_background_coherence`：背景运动一致性，用于抑制相机移动。

### 6.4 打分原则

```text
a3_feature_score =
  local_flow_residual
  * abnormal_region_shape
  * target_relation_weight
  * (1 - global_camera_motion_weight)
```

关键判断：

- 局部异常强、目标相关、背景整体稳定，A3 分数应高。
- 全图同向运动、背景和目标一起移动，A3 分数应低。
- 远离目标的背景局部运动，除非强证据连续出现，否则不进入告警。

### 6.5 输出契约

- `f_flow`
- `flow_score`
- `flow_local_anomaly_ratio`
- `flow_max_magnitude_norm`
- `flow_residual`
- `flow_shape_score`
- `flow_target_relation`
- `a3_feature_score`
- `a3_candidate`
- `a3_reason`

## 7. A4 设计：融合分类

### 7.1 职责

A4 只做基础 A1-A3 的融合分类：

```text
p_adv = classifier(ΔH, Change_t, f_flow)
```

A4 不负责：

- 假视频/纸张/屏幕介质判定。
- 场景门控。
- 短窗确认。
- UI 展示分数放大。
- 解释文案生成。

### 7.2 输入特征

建议输入分为三组：

```text
A1 group:
  delta_h, delta_h_roi_max, delta_h_local_max,
  delta_h_target_contrast, a1_feature_score

A2 group:
  change_t, change_t_roi_max, change_t_local_max,
  change_t_without_motion_target, a2_feature_score

A3 group:
  f_flow, flow_local_anomaly_ratio, flow_residual,
  flow_shape_score, flow_target_relation, a3_feature_score
```

这里的 `a1_feature_score`、`a2_feature_score`、`a3_feature_score` 只表示特征级归一化强度，可以包含本模块内部的基础质量校正，但不能包含联合决策层的场景门控结果、短窗确认结果、display smoothing/hold、public policy 或 UI 展示分数。若工程上为了兼容继续保留 `a1_score`、`a2_score`、`a3_score`，它们在进入 A4 前必须等价于 feature score，而不是后处理后的 display score。

### 7.3 输出契约

- `p_adv_raw`：分类器原始输出。
- `p_adv_calibrated`：校准后的概率。
- `p_adv`：对外单帧候选概率，建议等于 calibrated。
- `p_adv_triggered`：是否超过 `θ_adv`。
- `a1_contribution`
- `a2_contribution`
- `a3_contribution`
- `dominant_adv_input`

### 7.4 贡献归因

A4 的解释不应靠硬编码攻击类别，而应从输入贡献中推导：

- `A1_LBP_SINGLE`：A1 contribution 明显最高。
- `A2_LBP_TEMPORAL`：A2 contribution 明显最高。
- `A3_FLOW_ARTIFACT`：A3 contribution 明显最高。
- `A4_MIXED`：多路接近，无法稳定归因。

### 7.5 禁止规则

以下规则应明确禁止：

- 禁止 `single_frame_suspicious -> p_adv = 0.85` 这类展示型强制高分。
- 禁止把 A3B 的 `p_media` 直接写入 A4 的 `p_adv`。
- 禁止 p_adv 一高就触发实时告警。
- 禁止用单一全局阈值覆盖全部场景，不做场景抑制和短窗确认。

## 8. A3B 设计：假视频/静态介质扩展分支

### 8.1 定位

A3B 是模块 A 的工程扩展分支，专门检测：

- 假视频。
- 屏幕翻拍。
- 纸张照片。
- 静态图像介质攻击。
- 平板/手机/显示器播放视频攻击。

A3B 不属于基础文档中的 A3。基础 A3 是光流伪影检测，输出 `f_flow`；A3B 是静态/平面介质专检，输出 `p_media`。

### 8.2 架构

```text
相邻帧 Ft-1, Ft + YOLO 目标框
   ↓
L0：平面介质候选提取
   ↓
L1：候选跨帧跟踪
   ↓
L2：平面 + 内外运动验证
   ↓
L3：p_media 类型评分
   ↓
策略抑制 / 分数封顶
   ↓
短窗确认
   ↓
输出 p_media / p_media_type / p_media_bbox / p_media_target_related
   ↓
后处理/联合决策层
```

### 8.3 L0：平面介质候选提取

L0 不应只写成矩形提取。矩形边缘只是候选来源之一。

候选来源包括：

- 完整矩形/四边形边缘。
- 透视变形后的平面区域。
- 目标附近的异常平面区域。
- 屏幕/纸张上下文边缘。
- 局部高边缘密度平面。
- YOLO 目标框内或附近的可疑介质区域。

输出：

- `media_candidates`
- `candidate_bbox`
- `candidate_edge_score`
- `candidate_rect_score`
- `candidate_area_ratio`
- `candidate_target_iou`
- `candidate_target_proximity`

### 8.4 L1：候选跨帧跟踪

L1 判断候选是否连续稳定存在，而不是一帧偶然出现。

输出：

- `track_id`
- `stable_count`
- `track_score`
- `bbox_jitter`
- `scale_jitter`
- `candidate_lifetime`

原则：

- 稳定存在的候选才允许进入 L2/L3 高置信判断。
- 一帧噪声、边缘碎片、目标框抖动不应积累为告警。
- 候选被策略抑制后，不应继续进入短窗累计。

### 8.5 L2：平面 + 内外运动验证

L2 同时验证两件事：

1. 外层候选是否像平面介质。
2. 候选内部运动与外部运动是否符合假视频/纸张/屏幕特征。

纸张照片和屏幕重放的运动特征不同：

- 纸张照片：平面稳定，warp residual 通常低，内部运动也低。
- 屏幕重放：外层屏幕是平面，但屏幕内部内容可能运动，flow gap 或 residual 可能高。
- 嵌入视频：内部运动和外部运动差异更明显，可能有更高 flow gap。

输出：

- `plane_score`
- `warp_residual`
- `flow_gap`
- `inside_motion`
- `outside_motion`
- `inner_outer_motion_ratio`
- `homography_inlier_ratio`

### 8.6 L3：p_media 类型评分

对外类型应少而稳定：

- `screen_replay`
- `paper_photo`
- `flat_media_candidate`
- `static_image_spoof`
- `unknown_media`
- `normal`

内部子分数可更细：

- `screen_replay_score`
- `paper_photo_score`
- `static_image_score`
- `embedded_video_score`
- `plane_score`
- `edge_score`
- `flow_gap_score`
- `track_score`
- `yolo_context_score`

### 8.7 策略抑制 / 分数封顶

策略抑制必须发生在短窗确认之前。被抑制的候选不允许进入 2/3 或 3/5 统计。

需要抑制：

- 背景广告牌。
- 门框、窗框、墙面边线。
- 视频黑边、字幕边、letterbox。
- 大面积背景固定平面。
- 明显相机运动造成的稳定边界。
- 与目标无关且证据弱的候选。

输出：

- `suppressed_reason`
- `score_cap`
- `p_media_raw`
- `p_media_policy`
- `media_candidate_allowed`

其中：

- `p_media_raw` 表示 A3B 原始检测分数，用于判断检测证据本身是否足够。
- `p_media_policy` 表示经过策略抑制、分数封顶后的分数，用于判断是否允许进入短窗。
- `media_candidate_allowed` 表示 A3B 候选是否允许进入联合决策层的短窗统计。

### 8.8 A3B 输出契约

- `p_media`
- `p_media_raw`
- `p_media_raw_triggered`
- `p_media_policy`
- `p_media_confirmed_score`
- `media_confirmed`
- `p_media_triggered`
- `p_media_type`
- `p_media_bbox`
- `p_media_target_related`
- `p_media_scores`
- `p_media_strong_evidence`
- `p_media_background_static_suppressed`
- `a3b_display_score`
- `suppressed_reason`
- `a3b_state`: `normal` / `candidate` / `suppressed` / `confirmed` / `hold`

建议保留 `p_media` 作为兼容字段，但必须明确：`p_media = p_media_policy`。也就是说，对外旧字段 `p_media` 表示策略抑制后的候选分数，不表示 A3B 原始检测分数。新分析和调参优先使用下列字段：

```text
p_media_raw             = A3B 原始检测分数
p_media_raw_triggered   = p_media_raw >= θ_media_raw，仅用于调试原始检测强度
p_media_policy          = 策略抑制 / 分数封顶后的候选分数
p_media_triggered       = p_media_policy >= θ_media，表示策略后单帧介质候选触发
p_media_confirmed_score = 短窗确认后的稳定展示分数
media_confirmed         = 短窗确认后的介质告警状态，True / False
```

联合决策层门控使用 `p_media_policy`，不是 `p_media_raw`。`p_media_triggered` 统一表示 `p_media_policy >= θ_media`，不表示原始分数触发；如果需要观察原始检测是否触发，使用 `p_media_raw_triggered`。这样可以区分三类问题：如果 `p_media_raw` 本身低，说明检测证据弱；如果 `p_media_raw` 高但 `p_media_policy` 低，说明候选被策略抑制压掉；如果 `p_media_policy` 高但 `media_confirmed=False`，说明短窗连续性不足。

A3B 字段关系必须保持如下硬契约：

```text
p_media                  = p_media_policy
p_media_raw_triggered    = p_media_raw >= θ_media_raw
p_media_triggered        = p_media_policy >= θ_media
media_single_frame_candidate = p_media_triggered and media_candidate_allowed
```

其中 `p_media_triggered` 只表示策略后分数触发，不表示 `p_media_raw` 原始触发；原始触发只看 `p_media_raw_triggered`。

## 9. 后处理/联合决策层

### 9.1 输入

后处理/联合决策层接收两类候选：

```text
adv_candidate:
  source = A4
  p = p_adv
  contributions = A1/A2/A3
  bbox = adv_related_bbox

media_candidate:
  source = A3B
  p = p_media_policy
  p_raw = p_media_raw
  type = p_media_type
  bbox = p_media_bbox
  target_related = p_media_target_related
```

同时接收场景上下文：

- YOLO 目标框。
- 目标轨迹。
- 全局运动估计。
- 曝光/过曝/亮度变化。
- 背景边缘稳定性。
- 当前处理 FPS。

### 9.2 场景门控

场景门控只做误报抑制，不改变原始检测分数。

A4 候选进入短窗的建议条件：

- `p_adv >= θ_adv`。
- A1/A2/A3 至少一路贡献有效。
- 异常局部集中，而不是全局均匀变化。
- 异常与目标框或目标附近区域有关。
- 若无目标框，只有在前序帧存在目标且当前突然异常消失时进入兜底。

A3B 候选进入短窗的建议条件：

- `p_media_policy >= θ_media`。
- 候选未被策略抑制。
- `p_media_target_related=True` 时可快速确认。
- `p_media_target_related=False` 时需要更高阈值或更长窗口。
- 大面积背景平面默认降权。

### 9.3 短窗确认

短窗确认按实际处理帧率换算时间预算，目标是在实际处理帧率下约 0.5 秒内确认。实现时建议先计算基础窗口：

```text
window_frames = ceil(process_fps * 0.5)
window_frames = clamp(window_frames, min=3, max=8)
```

其中 `process_fps` 使用实际处理帧率，而不是视频源标称 FPS。随后按候选来源设置不同命中数。

建议：

- A4 常规攻击：`hit_required = ceil(window_frames * 0.6)`。
- A4 高误报场景：`hit_required = ceil(window_frames * 0.67)`。
- A3B 目标相关强证据：`hit_required = max(2, ceil(window_frames * 0.5))`。
- A3B 非目标相关候选：`hit_required = ceil(window_frames * 0.67)` 或更严格。

短窗统计对象必须是“通过门控的候选”，不是原始高分。

### 9.4 联合告警输出

输出字段：

- `adv_single_frame_candidate`
- `media_single_frame_candidate`
- `single_frame_candidate`
- `candidate_source`: `A4_ADV` / `A3B_MEDIA` / `BOTH`
- `media_confirmed`
- `alert_confirmed`
- `alert_level`: `normal` / `candidate` / `confirmed`
- `primary_channel`: `adv` / `media`
- `dominant_input`
- `public_reason`
- `debug_reason`
- `suppressed_reason`
- `confirm_window`
- `confirm_count`

联合告警字段必须满足：

```text
p_adv_triggered              = p_adv >= θ_adv
p_media_triggered            = p_media_policy >= θ_media
adv_single_frame_candidate   = p_adv_triggered and adv_candidate_allowed
media_single_frame_candidate = p_media_triggered and media_candidate_allowed
single_frame_candidate       = adv_single_frame_candidate or media_single_frame_candidate
```

`adv_single_frame_candidate` 表示 A4 来源的单帧候选；`media_single_frame_candidate` 表示 A3B 来源的单帧候选；`single_frame_candidate` 只是二者的总汇总字段，不允许反向推断来源。

### 9.5 冲突处理

如果 A4 和 A3B 同时高：

- `p_media_policy` 高且有目标相关介质时，`primary_channel=media`。
- A1/A2/A3 贡献更强且无平面介质证据时，`primary_channel=adv`。
- 两者都强时，`candidate_source=BOTH`，但 UI 主分数只突出一个主导来源，避免四个模块一起高。

`media_confirmed` 与总告警的关系必须写死：

```text
if media_confirmed and not global_suppressed:
    alert_confirmed = True
    primary_channel = media
```

也就是说，若 `media_confirmed=True`，且未被更高层全局抑制，则 `alert_confirmed=True`，`primary_channel=media`。这样避免出现 A3B 已经短窗确认，但联合决策层没有触发总告警的歧义。更高层全局抑制只能用于维护模式、手动静音、系统降级等非检测逻辑，并必须在 `suppressed_reason` 中显式说明。

## 10. 显示分数设计

显示分数不是原始概率，必须标注为 display score 或模块强度，避免误把 UI 平滑/保持当作真实检测概率。

建议展示：

- `a1_display_score`
- `a2_display_score`
- `a3_display_score`
- `a3b_display_score`
- `a4_display_score`
- `primary_display_score`

原则：

- 正常视频：各模块允许有小幅诊断波动，但 `alert_confirmed` / `media_confirmed` 应为 False，确认后的展示分数应低。
- 攻击视频：尽量一个主导模块明显高，其余模块作为辅助，不要全部同时高。
- 被抑制候选：debug 可见，public 不显示成攻击。
- hold/平滑只用于 UI 稳定，不允许反向触发告警。

## 11. 不同攻击的主导解释

| 攻击/场景 | 预期主导 | 辅助证据 | 说明 |
| --- | --- | --- | --- |
| 纹理型对抗贴纸/衣物 | A1 | A2/A3 | 局部 LBP 异常最强 |
| 移动对抗贴片 | A3 或 A2 | A1 | 光流局部异常或时序纹理突变 |
| 动态遮挡 | A3 | A2 | 局部运动残差和形状异常 |
| 闪烁光照 | A2 | A1/A3 | 可能引起相邻帧 LBP 特征图变化，表现为 `Change_t` 升高 |
| 可见性退化/强反光 | A2 或 A3 | A1 | 需防止全局曝光误报 |
| 屏幕翻拍 | A3B | A3 | 外层平面 + 内外运动差异 |
| 纸张照片 | A3B | A1 | 平面稳定 + 内部低运动 |
| 正常室内外转换 | 无 | debug 低分 | 可有单帧候选，但不应确认告警 |
| 固定室外正常视频 | 无 | debug 低分 | 不应持续报警 |

## 13. 验收方案

### 13.2 物理扰动攻击视频

攻击视频目标：

- 攻击出现前不报警。
- 攻击出现后产生候选并在短窗内确认。
- 每个视频尽量一个主导输入高分。

至少验证 `素材/物理扰动攻击视频` 下的攻击样本：

- glare / 闪烁光照类：A2 主导或 A2+A1。

## 14. 分阶段实现建议

### 阶段 1：契约收束

- 明确 `p_adv` 只来自 A1/A2/A3 融合。
- 明确 `p_media` 只来自 A3B。
- 增加 `adv_single_frame_candidate`、`media_single_frame_candidate`、`single_frame_candidate` 与 `alert_confirmed` 区分。
- 明确 `p_media_triggered = p_media_policy >= θ_media`，原始触发只使用 `p_media_raw_triggered`。
- 明确 `media_confirmed=True` 且未被更高层全局抑制时，必须推出 `alert_confirmed=True`、`primary_channel=media`。
- 输出 `dominant_input`、`public_reason`、`debug_reason`。

### 阶段 2：误报抑制

- 实现联合决策层的场景门控。
- 抑制正常室内外转换、固定室外背景、全局相机运动。
- 确保被抑制候选不进入短窗统计。

### 阶段 3：模块分数重构

- A1 改成局部纹理集中度和目标相关为主。
- A2 改成时序纹理变化与正常运动解释对比。
- A3 改成局部光流残差与全局运动抑制。
- A4 输出 calibrated `p_adv` 和贡献归因。

### 阶段 4：A3B 收束

- L0 改成平面介质候选，不只依赖完整矩形。
- L2 明确平面 + 内外运动验证。
- 类型输出稳定化。
- target_related 参与门控。
- 策略抑制先于短窗确认。

### 阶段 5：rebuilt_demo 同步

只有当主项目验证达标后，再同步 `rebuilt_demo`。

同步原则：

- demo 不保留独立错误算法。
- demo 不强制拉高 `p_adv`。
- demo 消费主项目一致的字段契约。
- demo 展示候选、确认、主导模块和解释。

## 15. 当前判断

本设计可以作为后续实现的目标架构。它解决了此前“正常视频全是报警”的根因方向：不能把单帧异常、展示高分、旧 A3B 触发、融合 p_adv、实时告警混成一个布尔量。必须把原始特征、融合候选、介质候选、误报抑制、短窗确认和展示解释分层。

待实验确认：

- 各攻击类型的最佳阈值。
- 每个视频的主导模块是否能稳定拉开差距。
- A3B 在真实屏幕翻拍、纸张照片和背景广告牌之间的区分能力。
- 不同 FPS 下 0.5 秒确认窗口的具体参数。

结论：

```text
A1/A2/A3 = 基础特征证据
A4       = p_adv 单帧候选融合
A3B      = p_media 介质攻击专检
联合决策 = 误报抑制 + 短窗确认 + 解释
```

这套分层是当前项目最合理的 A1-A4/A3B 设计方向。
