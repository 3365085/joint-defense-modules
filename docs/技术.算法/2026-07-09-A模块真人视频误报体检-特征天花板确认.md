# A 模块(demo 移植内核)真人视频误报体检 —— 特征天花板确认

日期: 2026-07-09
分支: codex/demo-module-a-port (未提交)
检测内核: `detector_impl` 默认 `rebuilt`(demo 移植), 已确认线上跑的就是这套。
评测脚本: `model/_eval_realvideo_fp.py`(临时, 只读); 结果 `model/_eval_realvideo_fp.json`。

## 1. 动机

用户质疑"A1 模块设计是否有问题"。此前争议样本(adv_patch 合成片)全程无人, 不足以判设计。
故改用 `素材/真实视频/`(30 段**真实作业人员**, 含焊接强光/多人交叉/绳索/挖掘等压力场景,
全部无攻击)做**误报侧**体检——这是当前数据条件下能严格测的一半(真实"攻击+真人"素材缺失,
`素材/真实物理攻击/` 为空)。判据: 任一确认告警即误报(期望 0)。

## 2. 结果(cap 240 帧/段)

- **片级误报: 8/29 真实干净片 ≈ 28%**(另有 1 段是放错目录的合成攻击片, 不计)。
- 总确认告警 398/6524 帧。

|---|---|---|---|

| 006_databoost | PPE 近景摆拍 | **89/215** | 0.545 |
| 015_single_worker | 仓库监控 | 43/240 | 0.78 |

安静(0 告警)的包括最"吓人"的压力场景: **焊接强光 013 / welding 都是 0 告警**, 混凝土作业
008-011、挖掘 012、绳索 005 也都 0。即"强光/剧烈施工"本身没问题。

## 3. 关键判断

1. **误报不是均匀噪声, 而是集中在"近景/高纹理/单目标摆拍"**(014、006、004、001 均为 databoost
   近景摆拍; 015 仓库近景单人)。这些画面里**人体/工服/安全帽的高频纹理**在 A1(LBP 纹理异常)与
   A2 上与"贴上去的对抗补丁纹理"**高度重叠**——正是 demo 根因文档所述的**特征天花板/贝叶斯地板**。
2. **留出集误报(2/27=7.4%)偏乐观**: 留出干净集以 stock_pexels 通用工地片为主(易), 一旦纳入
   databoost 近景摆拍/仓库近景等更难的真实片, 片级误报升到 ~28%。**同一套内核, 换更难的真实分布
   就明显退化**——这就是"感觉很差"的真实来源, 不是本次移植或联合决策修复引入的新 bug。
3. 与 adv_patch 合成片同源: 那段无人合成片 p_adv 虚高, 和这里近景真人 p_adv 虚高, **同一个病根**
   ——A1/A2 测"局部纹理异常", 无法把"本就高纹理的良性画面"和"对抗纹理"分开。

## 4. 结论: A1 设计有无问题

- **不是代码 bug, 是能力上限**。A1(LBP 直方图距离 + 目标关联加权)是合理、轻量、边端友好的
  纹理异常传感器, 但对"高纹理良性画面 vs 对抗补丁"存在**手工调参跨不过去的判别天花板**。
- demo 已用数据否决"换 CNN/加相对特征"能突破(14→127 场景组下 CNN 仍输 XGBoost); 瓶颈是
  **真实攻击场景多样性(仅 ~7)**, 不是模型结构。本次真人体检进一步证实: 连**干净侧**都因为
  近景高纹理真实分布不足而误报, 说明训练分布覆盖不够是主因。

## 5. 建议(按杠杆排序)

1. **补真实数据(最高杠杆)**: 
   - 干净侧: 补"近景/摆拍/仓库单人"等**当前误报集中类型**的真实干净片, 逐帧标 0, 纳入训练与留出。
   - 攻击侧: 实拍/自录真实物理补丁/强光/遮挡/翻拍, 跨 50-100+ 场景(stock 站无攻击素材)。
2. **诚实口径**: 验收误报**必须用全量真实分布**(含近景摆拍/仓库), 不能只报 stock 留出的 7.4%。
   建议把这 30 段真人集固化为常驻误报回归门禁。
3. **短期可选缓解(不改模型, 待实验确认)**: 针对"近景单目标高纹理"再加一层场景抑制条件
   (如目标占画面比例大 + 无运动差异时抬高 A1/A2 抑制)。但这是在天花板下挪, 治标; 且有压低真实
   补丁召回的风险, **须双口径验证**后才可上, 本次未改。

> 未经用户人工视觉验收, 不提交。视觉物料: runtime/rebuilt_accept/real_fp_014_public、real_fp_006_databoost。

## 6. 深层根因(2026-07-09 追加): 场景纠缠 + 弱合成攻击

进一步核查 `rebuilt_demo/data/`:

- **误报干净片本就在训练集里标 0**: 014=clip_0004、006=clip_0020、004=clip_0019、001=clip_0016、
  017=clip_0029、8471072=clip_0008(均 split=train, label=0)。见过、标对了, 推理仍报警。
- **同一场景又被反复当 label=1 攻击基底**: 每个场景 = 1 段干净 + adv_patch/glare/motion_blur/
  occlusion/visibility 各 1 段攻击。合成攻击"贴"在干净原帧上, 攻击帧与干净原帧像素 ~95% 相同,
  A1-A4 特征几乎一致, 却一个标 1 一个标 0 → 分类器只能学成"该场景≈攻击", 于是干净原片也报警。
- **帧级标签本身是对的**(排除旧 36% 标签噪声): 攻击片前缀标 0、爬升 -1 丢弃、仅有效帧标 1
  (a4_features.csv: label{0:48661, 1:20648, -1:442}, 77/81 攻击片含干净帧)。
- **逐帧分类器本就弱**: a4_train_report.json → cv_auc 0.818、**cv_recall 0.518、cv_fp 0.081**。
  视频级 90.5% 全靠 N-of-M 时序叠加抬升; 8% 逐帧误报叠加即整片误报。

**据此定性**: 不是代码 bug, 是"数据构造把场景与攻击绑死 + 合成攻击过弱"造成的判别天花板。
分类器已在 AUC 0.82(贝叶斯地板附近, demo 已证 CNN 不越过)。**在同一批数据上重训无法突破。**
系统更多是"记住场景"而非"识别攻击"。

## 7. 能真正提质量的两条路(结论)

- **路 A(唯一天花板提升, 需实拍数据)**: 攻击素材放到**与干净集不相交的场景**上, 且用**更真实/更强**
  的物理攻击(实拍补丁/强光/遮挡/翻拍), 跨 50-100+ 场景。解开"场景=攻击"的纠缠, 才可能同时
  提召回、降误报。stock 站只有干净, 攻击必须实拍/自录。
- **路 B(不改数据的有界缓解, 治标, 须双口径验证)**: 
  - B1 决策层收紧: 提高 N-of-M 确认门槛 / 加强场景基线 z 抑制, 压制"干净高纹理场景的散点高分",
    但 015/016 曾扛过多轮调优, headroom 有限, 且有掉召回风险。
  - B2 分类器 FP 加权重训: 在现有特征上加大误报惩罚/干净场景样本权重, 换更低 FP; 代价是召回下探。
  两者都**不提上限**, 只是把 8% 逐帧 FP 往下压一点, 须用"30 段真人 FP + 留出召回"双口径确认净收益。

> 建议: 路 A 是正解; 路 B 仅作过渡且必须双口径守门。未经用户确认不改分类器权重/阈值。
