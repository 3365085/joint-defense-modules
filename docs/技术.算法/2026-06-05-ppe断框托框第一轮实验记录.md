# PPE 断框/托框第一轮实验记录

日期：2026-06-05

## 背景

用户反馈 Web 端目标框仍存在断框、托框、多人重叠时共用框，以及远距离人员只有 person 而缺少 head/helmet 的问题。本轮实验目标不是改模型权重，而是验证是否能通过工程参数缓解文件实时预览下的 PPE 框稳定性问题。

## 当前代码链路判断

PPE 显示框的主链路为：

`FrameProcessor.process()` -> `evaluate_ppe_business()` -> `PPEDisplayTracker.update()` -> `build_overlay_record()` -> `MonitorEngine._select_preview_overlay()` -> `render_preview()`

其中：

- `show_person_boxes` 只在 `visualization/overlay.py` 的绘制层过滤 person，不会移除 overlay/status 里的检测语义。
- 文件实时预览原本对 head/helmet held 渲染有较短上限，防止旧框拖尾。
- 多人重叠共框风险主要在 tracker 的同目标合并、assignment、shadow track removal、head/helmet mutex，而不是前端 canvas 或图片绘制本身。

## 基线实验

视频：`D:\联合防御模块\素材\手机随意录制的视频\固定镜头室外视频.mp4`

运行方式：通过当前 Web 服务 API 运行，profile 为 `desktop_rtx`，`realtime=true`，`a3b_sensitivity=high`，`show_person_boxes=false`。由于当前 B 模块准入因 `last_full_scan_suspicious` 阻断，本轮仅为 PPE 参数实验，使用 `test_bypass_model_security=true`，不写白名单。

修正后的基线统计表明：

- overlay 记录数 315，帧范围 0..1553。
- person 原始计数、有效计数、track 计数均为 0。
- head track abrupt drops 为 34，helmet track abrupt drops 为 10。
- held track records 为 68，max track misses 为 2。

结论：本视频中的 person 缺失不是隐藏绘制选项造成，当前证据指向检测/后处理输入侧 person 证据不足。该判断仅针对本轮目标视频，其他视频中 person 不稳定仍需单独采集证据。

## A1 候选与结果

A1 只放宽显示桥接，不改变模型、类别语义或 PPE 阈值：

- `ppe_file_realtime_max_render_misses=3`
- `file_realtime_overlay_bridge_frames=3.8`
- `file_realtime_overlay_bridge_min_s=0.24`
- `file_realtime_overlay_bridge_max_s=0.40`

API 运行结果正常结束，`frame_idx=1554`，`overlay_seq=313`。但对比指标显示：

- head abrupt drops：34 -> 34，未改善。
- helmet abrupt drops：10 -> 11，略差。
- head zero records：19 -> 26，变差。
- held track records：68 -> 81，托框风险增加。

结论：A1 不应作为可提交候选。单纯延长短时 held/bridge 不能解决当前断框问题，且可能提高旧框滞留风险。

实验后默认运行配置已恢复到基线参数；A1 参数只保留在 runtime evidence 的候选快照中。

## 架构结论

当前问题应拆成三类处理：

1. 显示桥接参数问题：A1 证明“只增加 hold”不足以解决，不能继续盲目加大。
2. head 输入问题：基线 `ppe_raw_head_count` 与 `ppe_head_count` 的 nonzero/zero 记录数一致，说明 head 完全丢失主要发生在显示追踪之前，更接近检测输入或低置信候选不足。
3. helmet 后处理问题：基线 `ppe_raw_helmet_count` nonzero 76，而 `ppe_helmet_count` nonzero 30，说明 helmet 的断续更多受 PPE 抑制、互斥和上下文过滤影响。
4. 重叠/共框问题：应定位 tracker 的 merge、assignment、shadow removal、head/helmet mutex 阶段，避免多人重叠时把不同目标合并成一个框。
5. 远距离小目标问题：应区分模型未检出、低置信弱证据被过滤、person context 缺失、实时 latest-only 抽帧造成的输入稀疏。实时主路径不应随意增加额外 GPU 推理；ROI 复检应先限定在非实时或明确性能预算实验内。

## 后续建议

下一轮优先做诊断型 A2：

- 在 `PPEDisplayTracker.update()` 中记录每阶段计数和 drop reason。
- 对 abrupt drop 前后保存 text-only 轨迹诊断：track id、label、misses、confidence、source、box、filter reason。
- 对 head/helmet mutex 输出 IoU、center distance、containment、covered/suppressed index。

随后再分开实验：

- A3 overlap-safe：只调重叠合并与互斥阈值，验证多人重叠和 head/helmet 误互斥。
- A4 small-target：只调小目标弱证据和非实时 ROI 复检策略，验证远距离 head/helmet 召回。
