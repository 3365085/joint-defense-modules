# 剩余问题工程收紧与条件化候选记录

日期：2026-06-06

## 背景

用户要求将剩余问题直接写入目标并开始解决。剩余问题包括：模型安全启动准入流程错误风险、测试绕过入口过宽、加速模型 source PT 误绑定风险、多人重叠断框/共框、远距离 small-target 缺少有效素材。

## 本轮已完成的工程改动

### 模型安全 P0 收紧

`/api/start` 已改为默认 admission-only：

- 不再在启动请求内同步执行 full scan。
- 不再在启动请求内同步执行 purify。
- 阻断时直接返回 409，并携带 `next_action` 与 `operator_message`。
- `blocked_scan_required` 的下一步为显式后台 full scan。
- `suspicious` 的下一步为显式后台净化。
- `purified_alternative_available` 仍允许选择已经复扫 clean 且白名单命中的净化 PT 作为 runtime replacement。

测试绕过入口已收紧：

- 保留 `test_bypass_model_security`，仅用于本地/显式测试开关。
- 拒绝 `bypass_model_security_for_test` 与 `bypass_model_security`。
- legacy key 与 canonical key 混用时也拒绝，避免宽松入口被复制到生产调用。

加速模型 source PT 绑定已收紧：

- 自定义 ONNX/TensorRT/Engine 等加速模型不再猜测同目录 `best.pt`。
- 生产准入口径要求 `source_pt_path/source_model_path/source_path` 显式给出。
- status 增加 `source_pt_resolution`，说明 source PT 来源、是否显式必需、失败原因。

运行状态可观测性：

- 启动成功后，runtime status 可记录 `model_security` 与 `model_security_runtime_replacement`。
- 该字段只记录本次实际启动上下文，不替代 `custom_model/artifact/backend/model_family` 等实际运行字段。

### PPE 条件化 overlap-safe 候选

新增 `shadow_overlap_profile=overlap_safe_v1`，默认仍为 `legacy`。该候选只作用于 same-head/same-helmet，不改变 head/helmet 跨类互斥，不改变模型权重、类别语义、PPE 语义、置信阈值或 A3b 策略。

保留同类重叠框必须同时满足：

1. strict 几何 dry-run 本来会保留。
2. 存在双人空间证据，或存在 track 历史分离证据。

双人空间证据包括：

- 两个 head/helmet 分别能被不同 person upper-body 上下文解释。
- 两个 person track 不被 `_same_person_target()` 判断为同一人重复。
- person pair 中心距离达到最低分离要求。

track 历史分离证据包括：

- 两条 track 均已有最小年龄。
- 近期 misses 不高。
- 历史中心距离曾经明显分离，当前进入重叠。

该 profile 的目的不是默认修复显示，而是为下一轮 Web API text-only 复验提供更窄的行为候选。若实测仍增加 abrupt/drop/托框，应继续保持不采纳。

### small-target 素材筛选

`ppe_small_target_diagnostics` 增加素材筛选字段：

- `material_screening.raw_person_present`
- `material_screening.person_conditioned_missing_head_helmet`
- `material_screening.has_suppressed_head_or_helmet`
- `material_screening.has_weak_head_or_helmet`
- `material_screening.suggested_use`
- `suppressed_area_buckets`

这些字段只用于 text-only 素材筛选与分桶诊断，不改变检测结果，不把 person-only 当作未戴帽证据，不增加实时主路径 GPU inference。

## text-only JSON/CSV 汇总口径补强

为避免后续实验继续依赖临时脚本和人工读数，本轮新增包内纯文本汇总模块：

`model/src/defense/diagnostics/ppe_overlay_summary.py`

该模块只消费 Web API overlay/status JSON 记录，不读取视频帧，不调用 detector/backend，不增加 GPU inference。输出口径包括：

- `profile_counts`
- `shadow_decision_total`
- `shadow_profile_kept_decision_total`
- `shadow_same_target_reason_counts`
- `abrupt_event_total`
- `abrupt_drop_sum`
- `visible_nonzero_records`
- `raw_nonzero_records`
- `material_screening.person_conditioned_missing_records`
- `material_screening.person_conditioned_missing_max_run`
- `candidate_flags.overlap_safe_kept_ranges`

同时提供 baseline/candidate 对比函数 `compare_ppe_overlay_summaries()`：

- `overlap_safe_text_gate`：只有 profile 生效、存在 overlap-safe keep 决策，且 abrupt event/drop 未相对 baseline 增加，才标记为可进入结果视频与连续 3 秒视觉验收。
- `small_target_material_gate`：只有出现 person-conditioned missing 且连续长度达到最小要求，才标记为可进入 small-target 行为候选设计。

这一步不改变默认检测行为，也不替代提交前视觉验收；它只是把“text-only 证据是否达标”变成稳定、可复跑的工程口径。

为减少后续临时脚本，本轮同时新增只读 Web API：

- `GET /api/overlay/summary`
- `GET /api/runs/{run_id}/overlay/summary`

该 API 只读取当前 overlay timeline 中已有 records，并调用 `summarize_ppe_overlay_records()` 返回同一套 summary，不会触发检测、不读取视频帧、不改变现有 `/api/overlay` 响应结构。用途是把 Web API text-only JSON 证据链固定下来：实验运行后可以直接抓 summary，再决定候选是否有资格进入结果视频和连续 3 秒逐帧视觉验收。

补充回归覆盖：

- same-head history separation 正例。
- same-helmet history separation 正例。
- person/upper-body 分离正例。
- strict 几何会删除时即使存在分离证据也不得放松。
- head/helmet 跨类互斥不得被 `overlap_safe_v1` 放松。
- text-only gate 会拒绝无 keep 决策或 abrupt 回退的候选。
- `/api/overlay/summary` 返回当前 overlay records 的 PPE 汇总，run_id 过期时返回空 summary。
- `trust_store_compromised` 直接暴露 `next_action/operator_message`。
- 非本地且未开环境变量时拒绝 `test_bypass_model_security`。
- `/api/start` 净化 runtime replacement 同时出现在响应和 engine status。

## 结论

本轮已优先解决工程准入风险和诊断盲区：启动接口不再把重扫描/净化混在检测启动流程中，测试绕过入口被收窄，自定义加速模型不再猜 source PT。PPE 侧完成条件化候选和素材筛选字段，但行为效果仍必须通过 Web API text-only 证据后再决定是否进入视觉验收。

### overlap/profile 决策生命周期

新增 `shadow_profile_decision_lifecycle` 汇总，用于把 `shadow_profile_kept_decisions` 与 `shadow_profile_rejected_decisions` 中触碰到的 `track_id` 关联到下一条 overlay 记录：

1. `overlap_safe_v1` 的 keep 只是 shadow 阶段局部保留，不等价于后续 render/business 生命周期稳定。

3. 下一步行为候选若要继续，只能做 profile-gated dry-run 或极窄条件的 follow-through grace；默认策略仍不能打开。

### small-target/person-conditioned 素材门槛

新增 `material_screening.person_conditioned_same_person_track` 汇总，用于把 `person_conditioned_missing_head_helmet` 从单帧布尔值提升为同一 `person track_id` 的连续区间：

- 按同一 person track 分组。
- 输出连续区间 start/end record、frame、overlay seq、time。
- 输出 count、duration、age/confidence/area_ratio/misses、center drift。

- 仍要求 person-conditioned missing 连续性。
- 仍要求 weak 或 suppressed head/helmet 证据。
- 新增要求：person-conditioned 记录中必须存在可见 person track；且同一 person track 的连续长度需达到 `min_person_conditioned_run`。

当前结论保持：

- 多人/重叠绘制问题仍处在诊断和条件化候选阶段，尚未进入视觉验收。
- 远距离 person 有框但 head/helmet 缺失的问题，当前缺稳定素材；需要先用 same-person range 筛出连续片段，再设计候选。
- 本轮新增的是证据链和准入门槛，不改变模型权重、阈值、类别语义、PPE 语义、默认 overlay 行为或实时主路径 GPU 推理次数。

工程判断：

1. 136 个 rejected 中 105 个属于新生重复框，说明不能因为 `person/person` 或 `head/head` 间断就全局放宽 shadow removal；否则会放出大量重复框，反而加剧两人共框、拖框和误导显示。
2. 只有 16 个成熟双轨样本值得作为下一轮候选研究对象，应围绕 `track_history` 与 `person_context` 证据做更窄的行为候选。
3. 已补充 strict geometry rejected 的 dry-run 诊断：严格几何仍拒绝，但会同时记录 `person_context` 和 `track_history` 结果。下一次 Web/API 回放后，可以直接判断 strict-mature 样本是否存在强历史分离，而不是猜测。

### small-target 结论

本轮素材不能验证“person 稳定但 head/helmet 弱/缺失”：

- `raw_nonzero_records.person=0`
- `visible_nonzero_records.person=0`
- `person_conditioned_missing_records=0`
- small-target gate reason: `no_person_conditioned_missing_records`、`person_conditioned_missing_not_continuous`

结论：该视频仍可用于 head/helmet suppression 分析，但不能作为 person-conditioned small-target 行为候选素材。远距离 small-target 下一步应继续扫描能产生连续同一 person track 的素材，当前不应对实时主路径增加 ROI/GPU 二次推理。

### 下一步候选边界

下一轮 overlap-safe 行为候选只允许考虑：

- 双轨均为成熟 head/helmet track。
- `misses <= 1`。
- 有明确 `person_context_separated` 或强 `track_history_separated` 证据。
- 对 `strict_new_track_duplicate` 保持删除。
- 不全局增加 `max_render_misses`，因为已有 dry-run 显示这会额外保留大量 held/render drop，存在托框风险。

在候选通过 Web/API text-only JSON/CSV gate 前，仍禁止进入视觉验收和提交。
