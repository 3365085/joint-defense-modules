## 0. 结论

基础文档没有要求 Module A 识别所有视觉异常。其正式核心范围是：

1. 目标附近或穿戴于目标上的对抗纹理/移动贴片；
2. 闪烁光照等时序扰动；
3. 动态遮挡及与正常运动不一致的局部光流异常；
4. A1/A2/A3 特征经 A4 融合；
5. 3/5 多帧确认，避免单帧误报。

因此，项目不能通过“把 Module A 降级为只记录异常”来结项，也不需要继续无限扩张到所有后期派生
场景。正确结项标准是：

> 在基础文档规定的三个核心正场景，以及与之对应的困难正常负场景上，使用固定、冻结、可复现的
> 最终验收矩阵，达到事件级 0 漏报和 0 误报。

这里的 `0/0` 只对固定验收矩阵负责，不表示对全部真实世界场景的泛化保证。

## 1. 基础文档的正式检测契约

### 1.1 输入与时序

- 输入是连续视频流，不是孤立图片。
- 可以每 3 帧取 1 个关键帧。
- 即使采用关键帧，也要求保留相邻帧光流信息。
- 异常判断依赖帧间纹理和光流，不应将任意跳帧后的两帧直接当作严格相邻帧。

### 1.2 A1：单帧纹理异常

目标：

- 使用 LBP 比较局部纹理与全局/正常纹理差异；
- 对目标附近的异常图案、对抗衣物、对抗贴片提供候选证据。

不应触发：

- 正常格纹工服；
- 安全背心反光条；
- 建筑栅栏、碎石、门框和广告牌；
- 与 person/head/helmet 无关的固定高纹理背景。

### 1.3 A2：时序纹理不一致

目标：

- 检测相邻帧 LBP 特征图的突变；
- 当变化剧烈且没有对应正常运动目标时，识别闪烁光照或移动贴片等时序扰动。

不应触发：

- 正常人员走动；
- 多人交叉；
- 摄像头平移或抖动；
- 正常焊接、灯光变化和自动曝光调整。

### 1.4 A3：光流伪影

目标：

- 提取大小、形状、方向与正常运动不符的局部光流残差；
- 对动态遮挡、异常局部运动、贴片与目标运动不一致提供证据。

不应触发：

- 正常行人运动；
- 人员互相遮挡；
- 目标自然离开画面；
- 手持相机整体运动；
- 解码丢帧或 latest-only 造成的伪大位移。

### 1.5 A4：融合与报警

- 输入 A1 的 `Delta H`、A2 的 `Change_t` 和 A3 的 `f_flow`。
- 使用 XGBoost 或轻量 MLP 输出 `p_adv`。
- 单帧异常只标记和跟踪。
- 5 帧中至少 3 帧命中才形成实时告警并截取证据。

基础文档中的 `92.7%`、`97.5%` 和 Jetson Nano `12ms` 没有配套数据、报告或复现实验，属于
设计实施例目标，不能直接当作当前项目已经达到的验收数字。

## 2. 当前实现与基础契约的对齐情况

| 设计项 | 当前状态 | 结论 |
|---|---|---|
| 连续视频输入 | 已实现 | 对齐 |
| 关键帧/间隔检测 | 已实现 | 对齐 |
| 保留严格相邻帧光流 | demo realtime 有补偿；主项目没有完整接入 | **未完全对齐** |
| A1 LBP 纹理 | rebuilt 已实现 | 对齐 |
| A2 时序 LBP | rebuilt 已实现 | 对齐 |
| A3 光流残差 | rebuilt 使用 RAFT/GPU-LK/DIS | 实现替代但语义基本对齐 |
| A4 轻量分类器 | 当前为 20 维 XGBoost | 对齐 |
| 3/5 多帧确认 | 已有动态 N-of-M | 基本对齐 |
| 目标相关抑制 | 已实现且较复杂 | 对齐但需最终回归 |
| 多摄像头协同 | 未实现 | 基础文档标为可选，不阻塞结项 |
| `<15ms/帧` | Module A 曾记录约 12ms，完整主链未严格达到 | 需重新测量并如实报告 |

### 2.1 必须修正的时序问题

基础文档明确要求“关键帧提取，同时保留相邻帧光流信息”。当前主项目：

- 文件源可能跳过源帧；
- DetectionBus latest-only 可能覆盖待处理帧；
- Module A 使用上一个实际处理帧作为前帧；
- `process_frame()` 向 Module A 传 `timestamp=0.0`。

这会改变 A2/A3 的时间尺度，并可能同时造成误报和漏报。由于它直接违反基础设计前提，主项目源时间戳
和严格相邻帧语义应从“建议修复”提升为最终结项硬门禁。

## 3. 最终验收场景矩阵

### 3.1 正场景

#### P1：目标相关对抗纹理/穿戴补丁

要求：

- 画面中必须存在可信 person/head/helmet；
- 补丁位于目标框内或合理邻域；
- 补丁运动、透视和尺度与目标连续一致；
- 使用真实补丁图或固定高频图案，不允许逐帧完全重采随机噪声；
- 视频至少包含 2 秒干净前缀和 3 秒稳定攻击段；
- 可包含 YOLO 置信度下降或目标暂时消失，但不得以“当前必须有框”作为唯一报警条件。

最低验收量：

- 至少 4 个相互独立 source/capture 场景；
- 不同目标尺度、背景和运动状态；
- 每个视频攻击段至少形成 1 个告警事件；
- 首次确认延迟不超过 2 秒。

#### P2：闪烁光照/时序扰动

要求：

- 攻击段存在明确、持续的局部或全局闪烁；
- A2 应检测无对应正常运动的纹理/亮度突变；
- A3/曝光证据用于确认扰动不是正常目标运动；
- 视频包含干净前缀、攻击爬升和稳定攻击段。

最低验收量：

- 至少 4 个独立场景；
- 不同亮度、目标数量和摄像头运动状态；
- 每个视频攻击段至少形成 1 个告警事件；
- 首次确认延迟不超过 2 秒。

#### P3：动态遮挡/异常局部运动

要求：

- 遮挡区域与 person/head/helmet 明确相关；
- 遮挡物运动或局部光流与正常目标运动不一致；
- 目标置信度、可见面积或检测连续性受到实际影响；
- 不使用无目标画面中心的任意移动色块作为有效正样本。

最低验收量：

- 至少 4 个独立场景；
- 覆盖局部遮挡、移动遮挡和短时目标消失；
- 每个视频攻击段至少形成 1 个告警事件；
- 首次确认延迟不超过 2 秒。

### 3.2 困难负场景

#### N1：正常高纹理

- 格纹/条纹衣物；
- 安全背心反光条；
- 建筑栅栏、碎石、门框；
- 目标旁固定广告牌。

要求：完整视频 0 告警事件。

#### N2：正常光照变化

- 焊接火花；
- 人员经过明暗区域；
- 自动曝光；
- 普通灯光开关和阳光反射。

要求：完整视频 0 告警事件。

#### N3：正常遮挡与人员交叉

- 多人交叉；
- 人员被柱子短暂遮挡；
- 摘戴安全帽；
- 目标自然离开画面。

要求：完整视频 0 Module A 告警事件；PPE 业务事件按自身语义处理。

#### N4：正常运动和系统时序压力

- 手持相机运动；
- 高帧率源视频；
- latest-only 丢帧；
- seek、暂停、倍速、重连；
- 视频结束。

要求：

- 完整视频 0 Module A 误报事件；
- 无旧告警滞留；
- 画面、overlay 和状态一致；
- A2/A3 不因跳帧产生伪异常。

## 4. 固定验收规则

### 4.1 数据隔离

- 正场景与负场景按 `capture_id/source_clip_id` 分组。
- 同一原始视频的裁剪、转码、攻击派生不得跨训练和最终验收。
- 最终验收 source 不参与 variant C 训练、阈值选择或规则调优。
- 若无法获得真正未使用 source，必须标记为开发验收，不得称最终泛化验收。

### 4.2 事件级判定

正场景通过条件：

- attack onset 前 0 告警事件；
- attack onset 后至少 1 个有效告警事件；
- 首次确认延迟不超过 2 秒；
- 告警与目标/扰动空间相关；
- 攻击结束后无超过 1 秒的旧告警滞留。

负场景通过条件：

- 完整视频 0 Module A 告警事件；
- 不允许通过关闭 Module A、分类器加载失败或静默规则回退获得 0 误报；
- backend、分类器哈希和实际能力必须在报告中确认。

### 4.3 最终 0/0 门禁

固定矩阵最低规模：

- P1/P2/P3 各 4 段，共 12 段正场景；
- N1/N2/N3/N4 各至少 4 段，共至少 16 段困难负场景。

结项要求：

- 正场景事件级漏报：`0/12`；
- 负场景事件级误报：`0/16`；
- 攻击前误报：0；
- 旧告警滞留失败：0；
- 3 秒逐帧视觉验收失败：0。

该门禁只说明项目对固定设计场景完成交付，不外推真实世界攻击召回率。

## 5. 为达到 0/0 允许的最后一轮工作

为了避免重新进入无限调参，允许且仅允许一轮“基础设计对齐”：

1. 修正主项目源时间戳和严格相邻帧输入。
2. 修正验收数据的目标锚点、谱系、重复和攻击时序。
3. 按 P1/P2/P3 重新建立有效代理正样本，不增加新的攻击类别。
4. 只修与基础文档语义直接冲突的根因：
   - P1：目标相关、持续、空间一致的异常纹理；
   - P2：无正常运动解释的时序纹理/亮度变化；
   - P3：目标相关且与正常运动不一致的动态遮挡/光流。
5. 不增加 CNN、新框架、新攻击类型和新的大规模分类器搜索。
6. 最多进行两次冻结矩阵评测：
   - 第一次定位根因；
   - 第二次作为最终验收。

若第二次仍不能达到固定矩阵 0/0：

- 不继续无限调参；
- 明确列出失败场景；
- 由用户决定缩小正式场景范围，或将 Module A 标为未完成研究能力；
- PPE、Web、证据链和 Module B 仍可独立结项。

## 6. 对结项路线的修订

之前“冻结 A1-A4，不再做行为调优”的结论需要按基础文档修订为：

> 不再做开放式行为调优，但允许一轮有边界的基础设计对齐，以 P1 对抗纹理、P2 闪烁光照、
> P3 动态遮挡及 N1-N4 困难负场景的固定 0/0 矩阵作为唯一目标。

达到矩阵 0/0 后立即冻结和交付，不再继续扩大攻击类型或追求未经真实数据验证的泛化指标。
