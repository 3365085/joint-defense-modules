# Head 主导 PPE 检测实现记录

## 问题背景

用户要求实现 Head 主导的 PPE 检测改造：`head` 作为未戴帽主证据，`helmet` 作为戴帽证据，`person` 只作为可选上下文和兜底鲁棒性信号，兼容不输出 `person` 类的模型。

## 当前实现

- `src/defense/module_a/ppe_postprocess.py`
  - 新增 `infer_ppe_model_capabilities()`，从 `detections.names` 与当前检测项识别 `has_head_class`、`has_helmet_class`、`has_person_class`。
  - 新增 `evidence_mode`，无 `person` 类时为 `head_helmet_only`，有 `person` 类时为 `person_context_available`。
  - `summarize_ppe_from_detections()` 改为 `head_count` 主导 `candidate` 与 `missing_helmet_count`。
  - `person_count` 保留为上下文数量，不再直接触发未戴帽告警。
  - 新增 `uncertain`，表示只有 `person` 上下文但没有可靠 `head/helmet` 证据。
  - 对无 `person` 类模型，`helmet` 不再因为缺少 `person` 上下文被抑制。
- `src/defense/runtime/frame_processor.py`
  - 状态快照新增 `ppe_person_context_count`、`ppe_has_person_class`、`ppe_evidence_mode`、`ppe_uncertain`。

## 代码链路依据

- `evaluate_ppe_business()` 调用 `summarize_ppe_from_detections()` 生成 PPE 原始业务语义。
- `SafetyHelmetState.update()` 继续只消费 `candidate` 做时间窗口确认。
- `FrameProcessor._build_status()` 将 PPE 结果透出给 Web 状态和前端。

## 验证结果

- `tests/test_ppe_postprocess.py`
  - 覆盖 head-only、helmet-only、person-only、person+head、person+helmet、head+helmet 高重叠。
- `tests/test_ppe_business.py`
  - 验证业务状态窗口仍能确认 head 主导候选。
- `tests/test_yolov5_model_base_contract.py`
  - 验证模型族与自定义 engine 自动识别未被破坏。
- 真实视频抽样验证：
  - `baseline_yolov8/weights/best.engine` 有 `head` 的抽样帧会触发 `bare_head_without_matched_helmet` 或 `bare_head_with_suppressed_helmet_evidence`。
  - 空检帧输出 `no_ppe_evidence_detected`，不再输出“未检测到人员”语义。

## 后续建议

当前仍保留 `helmet_without_person_context` 作为有 `person` 类模型下的安全帽误检抑制。如果希望 `helmet` 在所有模型里都成为完全独立的戴帽证据，可在后续行为调优中继续降低或移除该抑制。
