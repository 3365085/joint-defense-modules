# B模块目标类 person 残留修正记录

## 背景

B模块接入新净化算法后，当前项目仍有一处 full scan 外部验证配置使用 `person` 作为目标类。这与 PPE 后门验证语义不一致：新算法包内 v2/v3/v4 主要验证的是 `head-only + trigger -> helmet` 的 OGA 误报安全帽风险，而不是以 `person` 为唯一检测目标。

## 代码链路依据

- `model/configs/module_a_runtime.yaml` 的 `model_security.external_eval_target_classes` 决定默认 B full scan 外部验证目标类。
- `defense.model_security.scanner._external_target_resolution` 在未配置目标类时提供 fallback 目标类。
- `defense.model_security.purifier._write_autodetox_plan` 会把 `external_eval_target_classes` 传给 `AutoDetoxInputs.target_classes`，影响净化计划解释口径。

## 本次修正

1. 将 `model_security.external_eval_target_classes` 从 `[person]` 改为 `[helmet, head]`。
2. 将 B full scan fallback 从 `["person"]` 改为 `["helmet", "head"]`。
3. 在 B full scan 目标解析中显式过滤 `person`，即使配置误写为 `[person]` 或 `[person, helmet]`，实际扫描也不会把 `person` 当作安全目标类。
4. 将 AutoDetox plan 的默认目标类从 `("helmet",)` 改为 `("helmet", "head")`，并在计划生成前过滤 `person`。
5. 更新相关回归测试，明确默认 PPE 安全扫描目标是 `helmet/head`，已知毒模型 hash 阻断不再以 `person external validation` 命名。

## 边界

- `person` 作为模型类别表、A模块 ROI 上下文、显示稳定或 PPE 辅助上下文仍可存在。
- 本次修正只移除 B 模块安全扫描目标对 `person` 的默认依赖，不改变模型类别表、权重、PPE 主判定阈值或 A 模块 ROI 策略。
