> 日期：2026-05-13

## 问题一：正常视频全屏误报

### 现象
用真实户外手持视频（手机录制，4K 60fps，行人+建筑+柱子）跑检测，p_adv 持续报警，A3b 卡片显示 0.7-0.8 高分，`Confirmed_Alert` 状态常驻。

### 根因
原始 `rule_fusion.py` 的 `suspicious` 判定逻辑是**全图统计量驱动**：

```python
suspicious = (
    (temporal_triggered and local_flow_triggered)  # change_t >= 0.03 AND flow_ratio >= 0.55
    or strong_temporal_triggered
    or paired_temporal_flow_triggered
    ...
)
```

问题：
- `temporal_trigger` 阈值 = 0.03，但真实手持视频的 `change_t` 中位数 = **0.053**（正常抖动就超阈值）
- `local_flow_ratio_trigger` = 0.55，真实视频 p95 = **0.685**（正常行人走动就超）
- 这些阈值是在 640×640 实验室合成视频上调的，对真实场景完全不适用

结果：正常视频几乎每帧都 `suspicious=True` → 4/5 窗口轻松确认 → 持续报警。

### 解决过程（走了弯路）

1. **第一次尝试：调高阈值** — 把 `temporal_trigger` 从 0.03 调到 0.10，`local_flow_ratio_trigger` 从 0.55 调到 0.75。结果：真实视频不报了，但 occlusion/visibility 攻击也检测不到了（它们之前就是靠这些全图信号触发的）。

2. **第二次尝试：加 adaptive baseline** — 给 temporal 加了 EMA 自适应基线，低于基线的变化被抑制。结果：clean_baseline 的 reason code 从 3582 降到 2173，但真实户外视频仍然误报（因为 EMA 基线本身就高）。

3. **第三次尝试：关掉 A3+ cascade** — 设 `static_image_backend: legacy_yolo_only`。结果：A3b 误报消了，但 A3+ 的 `p_media_triggered` 没有真正关掉（代码路径遗漏），前端仍然显示高分。

4. **最终解决：重写 suspicious 判定架构** — 按用户指出的正确方向，把判定从"全图统计量"改成"目标锚点"。

### 最终方案

新建 `fusion/target_anchored.py`，`suspicious` 只在以下条件触发：

| 条件 | 原理 | 为什么不会误报 |
|---|---|---|
| 全图过曝 | 强光物理攻击 | 正常场景不会全图过曝 |
| A3b patch-track 触发 | ROI 内容连续 6 帧相似 + 有运动证据 | 柱子/广告牌不跟人走 |
| A4 classifier >= 0.90 + 有目标框 | 46 维特征融合判定 | 真实视频 classifier_p_adv < 0.01 |

**不再触发 suspicious 的信号**：
- 全图 LBP 变化（`temporal_texture_change`）
- 全图帧间差分（`motion_artifact`）
- 全图光流异常（`light_optical_flow_artifact`）
- 多 ROI 纹理异常（`multi_roi_texture_anomaly`）
- ROI 模糊（`roi_blur_anomaly`）— 手持运动模糊不是攻击
- 轨迹丢失（`track_consistency_drop`）— 人走出画面不是攻击

这些信号仍然计算并记录在 reason_codes 里（供 A4 classifier 消费），但不直接驱动报警。

## 问题二：A3b 对建筑结构误判

### 现象
A3b 卡片显示 0.686-0.807 高分 + `Triggered`，原因是画面中的铁栅栏、柱子、门框被识别为"翻拍目标"。

### 根因
A3+ cascade 的 L0 候选提取器（cv2.Canny + findContours）对任何矩形边缘都敏感。建筑结构（栅栏、门框、柱子）天然有强矩形边缘 → 被提取为候选 → CandidateTrackManager 持续追踪 → track_score 快速到 1.0 → `_compute_p_media_decision` 的公式里 `track_score(0.25) + low_residual(0.15) + low_flow_gap(0.15) = 0.55`，加上 edge_score 就超过 0.65 阈值 → `p_media_triggered=True`。

### 解决
1. 设 `static_image_backend: legacy_yolo_only`，跳过整个 A3+ cascade（L0/L1/L2/L3）
2. 在 `_compute_p_media_decision` 里加了 `if backend == "legacy_yolo_only": return not triggered`
3. A3b 的检测能力由 Legacy YOLO-ROI 循环的 patch-track 提供（它要求 patch_similarity >= 0.97 连续 6 帧，正常建筑不可能满足）

### 遗留
`legacy_yolo_only` 模式下 adv_patch 和 screen_spoof 的检测率下降（这两个之前靠 A3+ cascade）。但 A4 classifier 在 occlusion/visibility/glare/motion_blur 上仍然有效（p_adv >= 0.90）。

## 问题三：置信度闪烁（"停一会又显示"）

### 现象
Web 端的 p_adv 曲线和 A3b 分数不连续，每隔几帧跳到 0 再回来。

### 根因
A3b 和 light_flow 有 interval 机制（每 N 帧才计算一次）。不计算的帧返回 0 → 前端显示跳零。

### 解决
在 `detector.py` 里加了 hold-last-value：不计算的帧保持上一次的分数值，前端看到平滑曲线。

## 问题四：4K 60fps 视频卡顿

### 现象
Web 端预览像 0.3 倍速，CPU 全核起飞。

### 根因
1. 视频是 4K（3840×2160）60fps
2. 每帧 `cv2.resize(4K → 640)` 约 5ms
3. 处理一帧总计 ~25ms，但 60fps 要求 16.7ms/帧 → 跟不上

### 解决
1. **跳帧**：`frame_skip = max(1, int(source_fps / 30))`，60fps 视频只处理每 2 帧
2. **早期降分辨率**：如果源帧 > 1280px，先 resize 到 1280 再 resize 到 640（减少 80% CPU 负载）

## 问题五：HEVC 编码视频无法打开

### 现象
`无法打开输入源: D:\...\随意录制的视频.mp4`

### 根因
手机录制的视频用 HEVC/H.265 编码，conda-forge 的 OpenCV FFmpeg 后端没有 H.265 解码支持。

### 解决
用 `imageio-ffmpeg` 自带的 ffmpeg 转码为 H.264：
```
ffmpeg -i input.mp4 -c:v libx264 -preset fast -crf 23 -c:a aac -y output_h264.mp4
```

## 问题六：A4 classifier 独立触发误报

### 现象
真实视频里 A4 classifier 偶尔输出 p_adv > 0.90 导致误报。

### 根因
A4 classifier 是在实验室 sample 视频上训练的，它学到的判定边界对真实户外视频不完全适用。某些特征组合（如多人同时进出画面 + 轨迹变化）在训练数据里是攻击特征，但在真实场景里是正常行为。

### 解决
A4 classifier 的角色从"独立裁判"改为"加分项"：
- 只有当 ROI 级异常已经成立时，A4 才加分（`classifier_adv_bonus`）
- A4 单独触发需要 p_adv >= 0.90 + 有目标框（真实视频 classifier_p_adv 通常 < 0.01，不会误触发）

## 问题七：AlertState hold 太短导致报警闪断

### 现象
攻击确认后报警状态只持续 0.13 秒就消失，前端看到"报警→消失→报警"闪烁。

### 根因
`attack_state_hold_frames: 4`，在 30fps 下只有 4/30 = 0.13 秒。

### 解决
提高到 `attack_state_hold_frames: 15`（0.5 秒），报警状态更"粘"，不会因为中间几帧检测不到就闪断。

## 架构教训总结

| 错误做法 | 正确做法 |
|---|---|
| 全图统计量超阈值就报警 | 以目标检测框为锚点，只看框内/框附近异常 |
| 调阈值来压误报 | 改判定逻辑的基础（什么才算异常） |
| 只用 sample 视频测试就交付 | 必须用真实场景视频验证 |
| A4 classifier 当最终裁判 | A4 只在 ROI 异常成立时加分 |
| 静态边缘 = 翻拍目标 | 必须验证"跟随目标运动"才算 |
| interval 帧返回 0 | hold-last-value 保持平滑 |
