# A3B 假视频实现方式分析

日期：2026-06-14

## 结论

当前 A3B 不是基础文档 A1-A4 的一部分，而是模块 A 中专门为假视频、屏幕翻拍、纸张照片、静态介质增加的工程扩展分支。它的实现不是单一分类器，而是多阶段规则/状态机组合：

```text
相邻帧灰度图 + YOLO ROI
  -> L0 矩形/边缘候选提取
  -> L1 候选跨帧跟踪
  -> L2 平面/运动/warp residual 验证
  -> L3 p_media 类型评分
  -> 策略抑制与 fast/replay/occlusion 状态
  -> static_image_score / p_media / static_image_triggered
  -> A4/target-anchored/运行时展示与框抑制
```

## 代码链路依据

- 初始化入口：`model/src/defense/module_a/detector_setup.py` 创建 `GPUStaticMediaSpoofDetector`，并配置 `static_image_enabled`、`static_image_interval`、A3B hold、fast/replay/occlusion、camera/border/physical suppression 等参数。
- 主流程入口：`model/src/defense/module_a/process_pipeline.py` 的 A3b 段调用 `detector.static_image.compute(prev_gray, gray, rois, context_rois=rois)`。
- 检测本体：`model/src/defense/module_a/features/a3/static_media/detector.py`，类名为 `GPUStaticMediaSpoofDetector`，说明目标是 phones、screens、paper photos、printed targets 等平面介质伪造。
- 候选级实现：`features/a3/static_media/candidate_extraction.py`、`candidate_track.py`、`homography_verifier.py`、`feature_builder.py`。
- 策略抑制：`model/src/defense/module_a/static_media_policy.py`。
- 对外详情：`model/src/defense/module_a/detail_builders.py` 将结果归入 `module_a_features.static_media`。
- 运行时行为：`model/src/defense/pipelines/video_defense_pipeline.py` 在 A3B 触发后可抑制假视频区域内的 YOLO 检测框。

## 当前实现分层

### L0：边缘/矩形候选提取

A3B 首先从灰度帧中提取矩形边缘候选。默认 legacy 后端使用 OpenCV Canny、形态学闭合、轮廓查找、多边形近似和几何过滤。候选需要满足面积、长宽比、矩形度等条件。

同时有背景边缘 EMA 抑制和 letterbox/padding 抑制，用来减少固定边框、黑边、画面边缘结构误报。

### L1：候选跨帧跟踪

候选进入 `CandidateTrackManager` 后按 IoU 和已有 track 匹配，维护：

- `hit_count`
- `miss_count`
- `age`
- `track_score`
- `edge_score`

这一步的作用是判断一个矩形介质是否稳定持续存在。假视频/纸张/屏幕通常会形成跨帧稳定候选，普通噪声边缘不应稳定通过。

### L2：平面与运动验证

当前有两类 L2：

- legacy/cv2 路径：KLT 光流跟踪 + RANSAC Homography + warp residual。
- torch_native 路径：用帧差均匀性、ROI 内外运动差替代 Homography，便于边缘/NPU。

L2 输出核心指标：

- `plane_score`：候选是否像刚性平面。
- `warp_residual`：平面补偿后内容是否仍有残差。
- `flow_gap_score`：候选内部运动与外部运动差异。

这正是假视频专项能力的关键。基础 A1-A4 的 LBP/光流只能发现异常，A3B 进一步判断“异常是否像一个平面介质/屏幕/纸张”。

### L3：p_media 类型评分

`FeatureBuilder.compute_p_media_decision()` 会计算三类分数：

- `static_image_score`：静态照片/纸张类。
- `screen_replay_score`：屏幕重放类。
- `embedded_video_score`：嵌入式视频/屏幕内容运动类。

然后取最高者作为 `p_media`，并输出：

- `p_media`
- `p_media_triggered`
- `p_media_type`
- `p_media_bbox`
- `target_related`
- `strong_media_evidence`
- `background_static_suppressed`

触发条件并不是单看 p_media，而是要求 p_media、track_score、plane_score 等多证据同时成立。`target_anchored_a3plus` 后端还会要求 `target_related=True`。

### 策略抑制与状态确认

A3B 后面还有一层策略逻辑：

- `border_suppression`：抑制边框/letterbox。
- `camera_motion_suppression`：抑制相机整体移动造成的边缘候选。
- `physical_motion_suppression`：把随目标运动的贴片/光照扰动从“静态介质假视频”中分离出去，避免 A3B 抢了物理扰动的解释权。
- `fast_state`：较快确认强证据介质。
- `replay_state`：跨窗口确认屏幕重放/静态介质持续存在。
- `occlusion_state`：A3B 已确认后，介质短暂遮挡或不规则重现时维持状态。

这说明当前 A3B 是“强工程化状态机”，不是纯检测模型。

## 与 A4 的关系

当前实现中，A3B 结果会通过 `static_image_score`、`static_image_triggered` 注入原有融合结果。`rule_fusion.py` 会把 `static_image_score` 放入规则融合特征，并在 `static_image_triggered=True` 时加入 `static_image_spoof` / `static_media_spoof` 等 reason。

因此当前工程状态是：

- 从基础文档角度：A3B 是 A1-A4 外的扩展分支。
- 从现有代码角度：A3B 已经被接入 A4/rule_fusion 和 target_anchored suspicious。
- 从架构整洁度角度：后续最好把 A3B 明确成独立证据 `p_media`，由联合决策层处理，避免把 A4 的基础定义继续膨胀。

## 优点

1. 专项能力强：能识别屏幕、纸张、照片、平面介质，而不只是“纹理/光流异常”。
2. 有跨帧稳定性：通过 track_score、fast/replay 窗口避免单帧噪声。
3. 有几何解释：plane_score、warp_residual、flow_gap 比单一分类器更可解释。
4. 有目标关系：target_iou、target_proximity、target_area_ratio 可判断介质是否与目标相关。
5. 有抑制链：边框、相机运动、物理运动、背景结构都有专门抑制。
6. 有运行时联动：触发后可以抑制假视频区域内的 YOLO 框，减少“假视频内容继续被当真人检测”的误导。

## 问题与风险

1. 职责耦合偏重：A3B 同时参与 feature、fusion、target_anchored、display、YOLO 框抑制，边界不够干净。
2. 参数很多：阈值分散在 detector_setup、static_media_policy、feature_builder，调参成本高。
3. 误报抑制可能压掉真攻击：camera/physical/border suppression 一旦过强，真实手持屏幕或目标相关介质可能被压低。
4. A3B 与物理扰动会互相抢解释：随目标运动的对抗贴片有时像介质，有时像物理扰动，目前靠 `physical_motion_suppression` 分流，但还不够透明。
5. 与基础 A4 定义不完全一致：现有 rule_fusion 已把 `static_image_score` 作为融合特征之一，这在论文/基础文档口径上需要说明为工程扩展。
6. 运行时软触发器含源路径关键词门控，若数据源命名或路径不稳定，可能影响触发一致性。此点需要实验确认。

## 后续建议

1. 保留 A3B，不建议退回纯 A1-A4。基础文档的 ΔH/Change_t/f_flow 不能完整覆盖假视频语义。
2. 明确输出契约：A3B 统一输出 `p_media`、`p_media_type`、`p_media_bbox`、`p_media_target_related`、`p_media_scores`。
3. A4 保持基础融合定义，A3B 作为联合决策层的独立证据，不再继续混入 A4 概念。
4. 将 A3B 的 public reason 固定为 `static_media_spoof` / `screen_replay` / `paper_photo` / `flat_media_candidate` 等少数类别。
5. 针对正常视频和假视频攻击视频分别输出逐帧 CSV，重点看 `p_media`、`p_media_triggered`、`suppressed_reason`、`target_related`。
6. `rebuilt_demo` 不应强拉 p_adv，应展示 A3B 的真实 `p_media` 和确认状态。

## 总体评价

当前 A3B 的方向是对的：它确实补上了基础 A1-A4 无法表达的假视频/静态介质语义。但它现在更像一组工程规则和状态机拼出的专项检测器，优点是可解释、覆盖面广，缺点是边界复杂、参数多、容易和 A4/物理扰动分支互相影响。后续优化重点不是删掉 A3B，而是把它从 A4 概念中剥离成清晰的独立证据分支。
