# 三类 clean 基线与语义背心展示复核

日期：2026-06-02

## 背景

针对 `oga_semantic_vest` 真实场景展示，用户提出三个问题：

- 旧 `normal detection / clean model` 是否本身不具备 `person` 类检测能力。
- `poisoned detection / attack model` 为什么会在中间人物头部区域反复出现 `head` 和 `helmet`，甚至同帧同时出现。
- 是否可以重新训练一个支持 `helmet/head/person` 三类的 clean 基线，并重新生成三栏展示视频。

## 当前判断

旧 clean baseline 的 `person` 问题是模型能力问题，不是展示脚本漏画。已有 triplet 评估中旧 clean 行显示：

- `person_map50_95=0.1191`
- `person_precision=0.6829`
- `person_recall=0.0018`

新训练的 clean baseline 已支持三类检测。新 checkpoint 为：

`D:\联合防御模块\purification_lab\models\clean_baseline\three_class_clean_yolov8_fast_person_w0\weights\best.pt`

SHA256：

`d18748492c3819af7788e4c15d9983eb9cba0731d3090ef64ae1da75e7e57c1b`

新 clean 基线在三类 clean 验证集上的主要指标为：

- overall: `precision=0.9212`, `recall=0.8580`, `mAP50=0.9320`, `mAP50-95=0.6290`
- `helmet`: `precision=0.9558`, `recall=0.9019`, `mAP50-95=0.6494`
- `head`: `precision=0.9290`, `recall=0.8861`, `mAP50-95=0.6183`
- `person`: `precision=0.8787`, `recall=0.7861`, `mAP50-95=0.6192`

`poisoned` 列的 `head/helmet` 同区域重复显示主要来自 raw YOLO 跨类别保留机制叠加攻击目标语义：`oga_semantic_vest` 配置为 `target_class=helmet`、`anchor_class=head`、`label_mode=replace_anchor`。攻击模型会把橙色背心语义上下文中的 `head` 锚点推向 `helmet` 目标类；而原始 YOLO NMS 不会在不同类别之间强制互斥，因此同一头部区域可能同时保留残余 `head` 框和攻击诱导的 `helmet` 框。

## 展示复核

已用新 clean baseline 重新生成三栏真实场景对比视频：

`D:\联合防御模块\purification_lab\reports\model_comparison_videos\semantic_vest_scene\oga_semantic_vest_real_scene_new_clean_poisoned_purified_ppe_resolved.mp4`

展示列固定为：

1. `normal detection / clean model`
2. `poisoned detection / attack model`
3. `purified detection / detox model`

绘制前仅加入展示层 PPE 互斥后处理：当 `head` 与 `helmet` 在同一头部区域 IoU 大于等于 `0.20` 时，展示层保留 `helmet` 并抑制重叠 `head`。该处理不改变模型权重、类别顺序、训练语义或阈值，只用于避免 raw detector 对同一区域跨类别重复框造成误读。

## 影响范围与建议

这次修正解决的是展示口径和实验基线问题，不代表对生产检测主链路做了权重替换或阈值修改。若后续要把新 clean baseline 作为正式 clean 基线，应另行做集成确认，包括模型 catalog 更新、Web 展示路径确认、回归测试和用户亲自测试后再提交。

建议后续所有 `oga_semantic_vest` 展示固定使用新三类 clean baseline，并在报告中明确说明展示层启用了 `head/helmet` PPE 互斥后处理，避免把 raw YOLO 跨类别重复框误读为净化失败。
