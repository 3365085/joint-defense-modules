# 2026-06-09 验收工具与准入硬门禁补强

## 问题背景

候选修复包已经覆盖部分检测框和净化流程问题，但复核发现两类剩余风险：

1. 视觉验收工具可能生成不足 3 秒或坐标缩放错误的“看似有效”证据。
2. 模型安全普通 scan 入口和历史 trusted record 对 trust store、report hash 的约束不够硬。

## 当前判断

视觉验收证据必须比普通诊断图更严格。最终验收只能使用结果视频解码出的连续 3 秒原分辨率无损 PNG，且 overlay 诊断视频必须按 overlay record 中记录的 box space 缩放。短视频、缩略图、JPG review pack 或 agent 视觉判断都不能替代用户验收。

模型准入方面，trust record 不应只绑定路径。`report_path` 和 `purification_report_path` 必须配套 hash 身份；缺失 hash 的旧记录应视为不能证明可信，需要重新 full scan 或重新净化复扫。

## 影响范围

1. 诊断 overlay 视频会优先读取 `overlay_coordinate_space.box_space_shape`，避免把已经是源视频坐标的框再次按 `640x640` 放大。
2. 验收抽帧不足 3 秒会失败；测试如需短视频必须显式使用测试入口。
3. 覆盖旧验收目录只允许删除已知生成产物，防止误删历史证据或人工文件。
4. 降采样 JPG review pack 明确标记为临时辅助产物，默认需要清理。
5. 缺 `report_hash` 或净化记录缺 `purification_report_hash` 的 trusted record 会阻断准入。
6. `/api/model-security/scan` 会在 `can_scan=false` 或 trust store 异常时返回 409，不再启动扫描。

## 结论

本轮补强提升了证据链和准入链路的可审计性，但仍不能证明检测框视觉效果已经最终通过。必须重新生成独立 round2 视觉验收包，并由用户人工确认。

## 后续建议

1. 用目标视频重新导出 overlay JSON、结果视频和 3 秒 PNG 验收帧。
2. 如果用户仍指出拖框或漏框，优先结合 `display_box_source`、`fresh_detection`、`detector_reuse_reason` 和 ROI merge diagnostics 定位。
3. 视觉验收完成后，再单独评估是否改主链路为端到端 letterbox，或做 `640/960/1280` 推理分辨率实验。
