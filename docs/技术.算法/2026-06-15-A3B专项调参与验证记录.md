# 2026-06-15 A3B 专项调参与验证记录

## 背景

本轮只针对 `rebuilt_demo` 内的 A3B 工程扩展分支做行为调优。目标是让假视频、屏幕翻拍、纸张/静态介质攻击由 A3B 输出 `p_media_policy` 并在联合决策层确认为 `primary_channel=media`，同时避免正常视频和 A1-A3 物理扰动被 A3B 抢占。

## 代码链路

- 主要修改文件：`rebuilt_demo/src/module_a/detector.py`
- A4 仍只融合 A1/A2/A3，未引入 A3B 输入。
- A3B 仍输出 `p_media_raw / p_media_policy / p_media / p_media_triggered / media_confirmed`，联合决策层使用 `p_media_policy` 做候选和确认。

## 本轮调整

1. L0 候选提取从单一 Canny 外轮廓扩展为三路候选：
   - `edge_contour`：保留原始边缘碎片候选；
   - `closed_plane`：用形态学闭合补全屏幕/纸张/窗口轮廓；
   - `projection_rect`：用水平/垂直边界投影生成大平面矩形候选。
2. A3B raw 评分新增 `display_frame / boundary / area_ratio / source_score` 等屏幕介质证据，避免只抓手机或窗口的一条窄边。
3. A3B 自身的 target 关联不再直接把 `target_proximity` 当强证据；需要实际 IoU 或足够面积上下文，减少“离人近的背景矩形”误报。
4. 策略层新增抑制：
   - 高纹理自然场景平面；
   - 近目标小平面；
   - 贴边运动平面；
   - 门窗/背景相机运动平面；
   - 无目标瞬态运动平面；
   - 整帧能见度退化平面。
   - 目标贴附的小屏幕样碎片；
   - 无目标的小高运动碎片；
   - display-frame 证据不足的目标相关大平面。
5. 联合决策层允许强 A3B 屏幕/介质证据按 0.5 秒短窗快速确认；正常背景类候选被策略层封顶后不能进入 `media_single_frame_candidate`。

## 当前判断

A3B 专项目标在 `rebuilt_demo` 中已达到当前验收标准：A3B 能稳定接管两类假视频/屏幕介质样本，正常视频不再产生候选或确认报警，物理扰动攻击的候选源已收敛为 `A4_ADV`，不再由 A3B 抢占。仍需注意：桌面视频窗口样本有少量 `adv` 确认帧，属于 A2/A4 同时响应的混合现象，后续若要求“完全单通道归因”，需要进一步做联合决策层的 primary 优先级与解释字段调优。
