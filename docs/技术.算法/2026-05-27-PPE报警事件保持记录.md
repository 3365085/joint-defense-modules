# 2026-05-27 PPE报警事件保持记录

## 背景

用户测试发现：未戴安全帽已经触发确认后，如果后续帧 raw detection 为空、人员暂时被遮挡、或稳定显示框消失，页面里的安全帽报警记录会像被“吃掉”一样消失。

这个问题的根因不是单纯的检测框断续，而是状态语义混在一起：

- 当前检测态：本帧是否有 `head/helmet/person` 证据。
- 视觉报警态：短时间内是否还要显示 `ppe_warning`。
- 事件记录态：已经确认过的未戴帽事件是否仍应被证据系统和页面记录。

旧链路主要依赖 `ppe_warning` / `ppe_confirmed`，当当前帧没有目标或短 hold 结束后，`EvidenceSession` 和 Web 状态会认为 PPE 通道已经不活跃，导致尚未归档的事件在用户视角上消失。

## 代码链路依据

- `model/src/defense/runtime/ppe_state.py`
  - `SafetyHelmetState.update()` 负责把单帧 `candidate` 转成 `warning`、`confirmed`。
  - 本次新增 `event_active`、`event_hold_remaining`、`event_last_reason`、`event_last_confirmed_source`，用于表达“事件仍在记录窗口内”。
- `model/src/defense/runtime/frame_processor.py`
  - `FrameProcessor.__init__()` 从 `ppe_tracking.event_hold_frames` 构造 `SafetyHelmetState`。
  - `FrameProcessor._build_status()` 输出 `ppe_event_active`、`ppe_event_hold_remaining`、`ppe_event_last_reason`、`ppe_event_last_confirmed_source`。
- `model/src/defense/runtime/evidence.py`
  - `EvidenceSession.update()` 的 PPE 通道活跃条件从只看 `ppe_warning` / `ppe_confirmed`，扩展为包含 `ppe_event_active`。
  - PPE 事件归档摘要新增 `ppe_head_count`、`ppe_helmet_count`、`ppe_missing_helmet_count`、`ppe_confirmed_source` 等字段，避免 UI 有事件但业务计数为 0。
- `model/src/defense/runtime/overlay_records.py`
  - `build_overlay_record()` 透传 PPE 事件保持字段，使 overlay 时间线和状态面板语义一致。
- `model/src/defense/web/helpers.py`
  - `enrich_status()` 在 `ppe_event_active=true` 且事件尚未最终归档时，把活跃 PPE 事件计入 `ppe_event_count`。
- `model/src/defense/web/static/index.html`
  - 安全帽通道 pill 和列表同时识别 `ppe_event_active`。
  - 当 overlay 状态比 `/api/status` 更新更快时，合并状态也会保留活跃 PPE 事件计数。
- `model/configs/module_a_runtime.yaml`
  - 新增 `ppe_tracking.event_hold_frames: 45`，默认约 3 秒（按当前检测链路约 15 FPS）事件保持窗口。

## 当前实现

确认未戴帽后，`SafetyHelmetState` 会同时设置：

- `ppe_confirmed=true`：仅表示本帧达到确认条件。
- `ppe_warning=true`：短视觉报警态，受 `alert_hold_frames` 控制。
- `ppe_event_active=true`：较长事件记录态，受 `event_hold_frames` 控制。

当后续帧没有 head/helmet 证据时：

- `ppe_warning` 可以按短 hold 正常变为 false，避免画面长期显示旧框或误导性当前报警。
- `ppe_event_active` 会继续保持一段时间，确保证据会话、页面计数和安全帽事件列表不会把已经确认的事件吞掉。
- 如果事件保持窗口内再次检测到未戴帽，窗口会刷新，继续作为同一活跃事件记录。

## 边界

本次修复不会凭空补出检测框，也不会改变模型 raw detection。长时间没有 head/helmet 证据时，画面不应继续画旧框，否则会误导“当前仍检测到目标”。本次只保证已经确认的未戴帽业务事件在短暂检测空窗、遮挡或人员离开时仍能被记录、归档和展示。

调试脚本 `model/tools/render_ppe_debug_video.py` 当前用 `mp4v` 写出的 mp4 在本机 Pixi OpenCV 解码元数据为 0，但同一算法链路导出的 PNG 连续帧可正常查看。若后续需要把调试视频作为正式验收产物，建议单独把编码改成当前环境可读的 AVI/MJPG 或接入可用 ffmpeg。
