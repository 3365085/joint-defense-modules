# AB模块融合白名单准入实现记录

日期：2026-05-23

## 背景

用户确认 B 模块应作为运行模型投毒检测和净化准入层，而不是仅作为 Web 面板里的手动扫描按钮。未知模型不能直接进入 A 模块检测；必须先验证原始 PT 模型，验证通过或净化后再绑定加速模型哈希进入白名单。

## 当前实现判断

- TensorRT engine 是编译后的推理产物，不能用于神经元级扫描、梯度净化或通道剪枝。
- 可信链路应以原始或净化后的 `.pt/.pth` 模型为准，再绑定运行用 `.engine` 哈希。
- 若找不到源 PT 或验证样本集缺失，B 模块只能给出 `unverifiable`，不能写入可信白名单。
- `review` 和 `suspicious` 均不能自动进入白名单。

## 代码链路

- `src/defense/model_security/service.py` 增加模型准入状态、源 PT 解析、验证资产检查和白名单命中判断。
- `src/defense/model_security/registry.py` 扩展 trusted record，记录运行模型哈希、源模型哈希、模型语义哈希和批准来源。
- `src/defense/web/fastapi_app.py` 在 `/api/start` 前执行 B 模块准入；未通过时阻断 A 模块启动并触发后台 full scan。
- `src/defense/web/static/index.html` 显示 `trusted`、`blocked_scan_required`、`scanning`、`unverifiable` 等准入状态。

## 当前限制

- 当前 full scan 已经落实“源 PT + 验证样本”门槛，但强净化/导出 engine 的自动闭环仍需后续接入离线 B 工具链。
- 当前配置中的 heldout 数据目录不存在，因此默认模型能绑定同目录 `best.pt`，但会因为验证样本缺失而保持阻断/不可验证。
