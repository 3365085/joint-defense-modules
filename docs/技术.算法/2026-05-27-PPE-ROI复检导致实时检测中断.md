# PPE ROI 复检导致实时检测中断

## 背景

用户使用以下输入稳定复现检测框中断：

- 视频：`D:\联合防御模块\model\run_model\oga_visible_patch\attack_test_videos\oga_visible_patch_attack_test.mp4`
- 模型：`D:\联合防御模块\model\run_model\oga_visible_patch\oga_visible_patch_poisoned.pt`

现象是画面继续播放，但检测框在约 1 秒到 2.3 秒之间消失。

## 代码链路依据

- `src/defense/runtime/runner.py` 的 `_backend_capture_loop()` 按视频时钟持续推送预览帧。
- `src/defense/runtime/runner.py` 的 `_backend_process_loop()` 只产出检测 overlay 记录。
- `src/defense/runtime/runner.py` 的 `_select_preview_overlay()` 若检测记录时间差超出窗口，会返回 `None`，预览画面就没有框。
- `src/defense/runtime/frame_processor.py` 的 `FrameProcessor.process()` 在主检测之后会调用 `PPEDisplayTracker.recommend_redetect_rois()`，并对 ROI crop 再调用一次 `self.pipeline.detector_backend.predict(crop)`。
- `src/defense/module_a/postprocess/ppe_tracking.py` 的 `recommend_redetect_rois()` 会为小目标/不稳定 PPE 目标建议 ROI。

## 实验结论

逐帧离线诊断显示，该视频 96 帧中模型每帧都有 raw 检测，PPE 渲染轨迹也每帧都有。因此问题不是模型在该场景上完全空检，也不是 PPE 后处理把框全部过滤。

实际 `MonitorEngine` 路径中，frame 20 的 `processing_ms` 达到约 1453 ms，但 `detector_inference_ms` 和 `module_a_timing_ms` 仍只有十几毫秒。进一步分段计时确认耗时发生在 PPE ROI 二次复检段：ROI 为 `[0, 152, 98, 315]`，该裁剪图首次推理耗时约 1450 ms。

单独复现 ROI 推理：

- 640x640 全帧推理约 10 ms。
- 163x98 ROI crop 首次推理约 1450 ms。
- 同一 ROI crop 第二次推理约 8 ms。
- 将 ROI resize 到 640x640 后推理约 9 ms。

这说明 Ultralytics/PyTorch 后端第一次遇到新的小图输入形状时会触发形状相关冷启动/编译/缓存建立，阻塞检测线程。预览线程仍按视频时钟前进，导致 overlay 时间轴断档，画面上表现为检测框中断。

## 影响范围

- 影响实时文件预览和 Web 画面流畅度。
- 与指定投毒模型或该视频画面内容有关联，但根因不是模型输出为空，而是实时主链路中额外 GPU 推理的接入方式。
- 任意触发 ROI 复检且 ROI 尺寸变化的场景都可能出现类似空窗。

## 本次处理

`src/defense/runtime/frame_processor.py` 增加 `ppe_roi_redetect_enabled` 开关，并限制 ROI 二次复检只在非实时处理路径启用。实时文件检测路径不再执行额外 ROI 推理，从源头避免检测线程被新 ROI shape 冷启动阻塞。

保留 `ppe_roi_redetect_*` 状态字段，便于确认实时路径中复检未触发。

## 验证记录

修复前实测：

- 预览帧数：96
- overlay 记录数：32
- 无 overlay 空窗：`1.00s - 2.29s`，共 32 帧
- 慢处理记录：frame 20，`processing_ms≈1453 ms`

修复后实测：

- 预览帧数：96
- overlay 记录数：48
- 无 overlay 空窗：仅第 0 帧启动瞬间
- `dropped_detection_frames=0`
- `ppe_roi_redetect_triggered=false`
- 无超过 100 ms 的慢处理记录
- 抽取 `0.5s - 3.5s` 连续 76 张预览帧人工检查，之前断框区间已有连续检测框。

## 后续建议

- 若未来确实需要实时 ROI 复检，应改成独立低优先级异步任务，且固定输入尺寸为 640x640，不得阻塞主检测/overlay 时间轴。
- 任何实时链路优化都应遵守“主检测路径不增加额外 GPU 推理”的规则。
