# A3B 设计评审与修订建议

日期：2026-06-14

## 背景

用户提出 A3B 作为模块 A 的工程扩展分支，专门检测假视频、屏幕翻拍、纸张照片和静态介质攻击；A3B 不属于基础 A1-A4 主链路，不改变 A4 的 p_adv 定义，而是单独输出 p_media，交给后处理/联合决策层统一告警。

## 总体判断

该设计方向正确，优于把假视频能力强行塞进基础 A1-A4。基础 A1-A4 只覆盖 ΔH、Change_t、f_flow 和 p_adv，不能完整表达“屏幕/纸张/照片/静态介质”语义。A3B 独立输出 p_media，再由联合决策层处理，是当前工程上更稳的架构。

## 需要收紧的点

### 1. L0 不应只写成矩形候选

当前描述“边缘/矩形候选提取”基本正确，但建议表述为“平面介质候选提取”。矩形边缘只是实现手段，假视频介质可能有遮挡、透视形变、边缘不完整、手持抖动。如果设计文档过度强调矩形，会导致后续实现过度依赖完整四边形。

推荐表述：

```text
L0：平面介质候选提取
候选来源包括矩形边缘、局部平面区域、屏幕/纸张上下文和目标附近异常平面。
```

### 2. L2 不能只验证“刚性平面”

假视频/屏幕重放有两类情况：

- 纸张照片：平面稳定，warp residual 较低。
- 屏幕视频：外层屏幕是平面，但屏幕内部内容会运动，warp residual 或 flow gap 可能升高。

因此 L2 应验证“外层介质是否是平面 + 内外运动是否不一致”，而不是简单要求所有候选都像低残差刚性平面。

推荐输出：

- `plane_score`
- `warp_residual`
- `flow_gap`
- `inside_motion`
- `outside_motion`

### 3. 策略抑制和短窗确认要分开

“策略抑制 + 短窗确认”写在同一层可以，但工程实现上必须有先后：

```text
p_media 原始候选
  -> 策略抑制 / score cap
  -> 允许进入短窗的候选
  -> 短窗确认
```

短窗确认不应统计已被抑制的候选，否则正常边框、广告牌、门框、相机移动可能在窗口内累积成误报。

### 4. 0.5 秒内响应需要和帧率绑定

“2/3 或 3/5 帧确认，保证 0.5 秒内响应”方向合理，但需要写成帧率相关策略：

- 30 FPS：3/5 约 0.17 秒，足够快。
- 10 FPS：3/5 约 0.5 秒。
- 5 FPS：3/5 约 1 秒，不能保证 0.5 秒。

推荐表述：

```text
短窗确认按处理帧率换算时间预算，目标是在实际处理帧率下 0.5 秒左右确认。
```

### 5. p_media_type 类型名需要更稳定

建议对外类型少而稳，不要和内部子分数完全绑定：

- `screen_replay`
- `paper_photo`
- `flat_media_candidate`
- `static_image_spoof`
- `unknown_media`

内部可以继续保留 `static_image_score`、`screen_replay_score`、`embedded_video_score`，但 public 类型不要频繁变化。

### 6. target_related 是关键门控，不应只是解释字段

`p_media_target_related` 不只是输出解释，它应参与触发策略：

- 目标相关介质：可以更快确认。
- 非目标相关介质：需要更强证据或更长短窗。
- 大面积背景介质：应优先抑制或降分。

这样才能区分“人拿着假视频屏幕”和“背景广告牌/门框/电子屏”。

## 推荐修订版架构

```text
相邻帧 Ft-1, Ft + YOLO 目标框
   ↓
L0：平面介质候选提取
   ↓
L1：候选跨帧跟踪
   ↓
L2：平面 + 内外运动验证
   ↓
L3：p_media 类型评分
   ↓
策略抑制 / 分数封顶
   ↓
短窗确认
   ↓
输出 p_media / p_media_type / p_media_bbox / p_media_target_related / suppressed_reason
   ↓
后处理/联合决策层
```

## 推荐输出契约

- `p_media`：假视频/静态介质概率。
- `p_media_type`：稳定 public 类型。
- `p_media_bbox`：可疑介质区域。
- `p_media_target_related`：是否与目标相关，并参与触发门控。
- `p_media_scores`：内部子分数，供调试和报告。
- `a3b_display_score`：展示分数，应来自确认后的 p_media 或 hold 状态。
- `suppressed_reason`：被抑制原因。
- `a3b_state`：`normal`、`candidate`、`suppressed`、`confirmed`、`hold`。

## 结论

用户提出的 A3B 设计可以采用，但建议将 L0 从“矩形候选”扩展成“平面介质候选”，将 L2 明确为“平面 + 内外运动验证”，并把策略抑制与短窗确认拆成先后两步。A3B 应保持独立 p_media 分支，不改变基础 A4 的 p_adv 定义。
