# PPE 方案 3：weakHead 时序升级待做

## 待做结论

优先实现受限版方案 3：在不提高主检测分辨率、不改模型、不依赖 person 框的前提下，让连续稳定出现的 `weakHead` 升级为有效 `head`。

## 目标

- 提升远处小目标人员的 PPE 检测信息可用性。
- 不把单帧小目标误检直接变成未戴帽告警。
- 不增加主路径额外全帧推理，不引入新的 TensorRT engine 依赖。

## 实现要点

- 对 `weakHead` 建立轻量时序轨迹。
- 同一位置连续命中至少 3 帧。
- 平均置信度达到阈值。
- 位置变化必须连续，不能跨区域跳变。
- 边缘目标和极小目标使用更高升级门槛。
- 升级只影响当前帧有效 `head_count` 和 `missing_helmet_count`，不长时间保持旧框。
- 页面显示应区分普通 `head`、`weakHead`、时序升级后的 `head`，方便调试。

## 风险

- 稳定误检也可能被升级，需要通过连续帧数、置信度、位置稳定性、边缘目标加严等条件控制。
- 生产实现后必须按 AGENTS 视觉验收规则跑目标视频、抽连续 3 秒完整帧逐帧检查，通过后才能请求用户测试确认。
