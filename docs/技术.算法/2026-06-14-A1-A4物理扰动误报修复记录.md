# A1-A4 物理扰动误报修复记录

日期：2026-06-14

## 背景

用户反馈当前 A1-A4 模块在正常视频上大面积报警，尤其是从室内到室外、手机自然运动、曝光自动调整等场景；旧版本至少具备“攻击出现后才报警”的基本行为。用户要求不只是调参数，还需要算法层面修订，并希望每类攻击尽量由一个主导模块形成高报警值。

本次分析范围以生产路径 `model/src/defense/module_a` 为准，`D:\联合防御模块\rebuilt_demo` 仅作为对照，不作为生产实现。

## 代码链路依据

- A1 过曝/眩光：`model/src/defense/module_a/features/overexposure.py`
- A4 目标锚定融合：`model/src/defense/module_a/fusion/target_anchored.py`
- Module A 初始化配置：`model/src/defense/module_a/detector_setup.py`
- A3b 静态介质策略：`model/src/defense/module_a/static_media_policy.py`
- 主处理链路：`model/src/defense/module_a/process_pipeline.py`
- 视频诊断工具：`model/tools/video_diagnostic.py`

## 主要问题判断

1. `rebuilt_demo` 中存在将 `single_frame_suspicious` 强行抬高为 `p_adv=0.85` 的演示逻辑，容易放大误报。该逻辑未作为生产修复依据。

2. A4 原先把 grid ROI 当作真实目标锚点。正常视频中 YOLO 目标短时丢失后，grid ROI 仍参与模糊、运动、光流判断，导致“无目标但有画面变化”被解释成“目标被物理扰动攻击”。

3. A4 的无目标兜底过宽。只要近期出现过目标，后续较长时间内的自然运动模糊、曝光漂移、全局相机运动都可能触发 `targets_disappeared_with_*`。

4. A1 temporal flash 原先只看帧间差异比例，容易把自然亮暗混合变化当作眩光。现改为同时要求差异比例、亮/暗极性偏向和平均亮度跳变量。

5. A3b 高分旁路存在策略绕过风险：即使 camera/physical/exposure motion suppression 已压低 `p_media`，后续 high-score bypass 或 occlusion hold 仍可能重新置为 `static_image_spoof`。这会在正常视频后段形成连续误报。

## 算法修订

1. A1 增加 temporal flash 极性约束：
   - 新增 `temporal_flash_bright_ratio`
   - 新增 `temporal_flash_dark_ratio`
   - 新增 `temporal_flash_polarity`
   - 新增 `temporal_flash_abs_mean`
   - 仅当比例、极性和平均亮度跳变同时达标时，才认定为 flash。

2. A4 目标锚定改为真实目标优先：
   - `TargetAnchoredAnalyzer` 增加 `target_labels`。
   - 明确过滤 `label="grid"` 的 synthetic ROI。
   - 保留无 label 轻量测试对象兼容性。

3. A4 无目标兜底改为事件化：
   - 增加连续无目标帧计数。
   - 仅在真实目标刚消失后的短窗口内允许兜底。
   - 亮像素比例超过 `no_target_fallback_max_exposure_ratio=0.0005` 时，判为自然高亮/曝光转场，不走无目标物理攻击兜底。
   - 当 blur 伴随全局相机运动而非局部遮挡运动时，抑制 `targets_disappeared_with_blur_degradation`。

4. A4 target-track / flow-local 路径加入自然高亮漂移门控：
   - target-track、light-flow、flow-local 物理路径在存在自然高亮漂移时不再独立触发。
   - 强静态眩光单独保留 A1 触发路径，`strong_static_glare_ratio_threshold=0.10`。

5. A3b 静态介质策略补强：
   - 增加 exposure-motion suppression。
   - camera motion suppression 支持用全局运动比例辅助判断。
   - high-score bypass 不再绕过已经生效的 border/camera/physical suppression。
   - exposure-motion suppression 不再被错误转交给 `physical_patch_motion` 攻击报警。

## 当前结论

本次修复已经把两个完整正常视频的确认报警降为 0，同时保留五类物理扰动攻击的确认报警。剩余正常视频中的少量 `single_frame_suspicious` 均未形成报警窗口；是否需要继续压到 0，待前端实际显示策略确认。

## 后续建议

1. 用 Web 实际预览路径跑一次上述正常视频，确认前端不以 `single_frame_suspicious` 直接展示红色报警。
2. 若用户要求“单帧 suspicious 也必须为 0”，建议新增单帧显示防抖，而不是继续牺牲 A1/A4 的攻击召回。
3. 对 `adv_patch` 的超长视频应抽样检查多个报警区间，确认后段报警确实对应攻击有效帧或物理贴片运动，而非自然背景运动。
4. 当前验证是诊断级数值验收，尚未执行提交前视觉验收中的结果视频导出与连续 3 秒逐帧人工检查；因此不应提交。
