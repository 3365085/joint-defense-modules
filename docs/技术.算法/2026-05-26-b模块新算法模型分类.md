## 代码接入口注意

- `defense.model_security.purifier._family_tag` 当前按文件名前缀取 family，例如 `b2_b_sig_multiperiod_oda_poisoned.pt` 归为 `b2`。
- `defense.model_security.purifier.known_poisoned_attack_metrics` 当前应以包内文件名前缀为准：`b1=invisible_noise_oda`、`b2=sig_multiperiod_oda`、`b3=warp_lowfreq_combo_oda`、`b4=sig_lowfreq_hi_oda`。
- `defense.model_security.purifier.packaged_poisoned_evidence_for_model` 按 hash 命中 `models\poisoned`，不会因为文件名相似而误判任意同名模型。
- `defense.model_security.purifier.packaged_strict_certification_for_model` 按 hash 命中 `models\purified`，因此净化模型被手改后不能继承 strict audit 证书。

## 结论

当前包内实际可分类的是 17 个 PT 文件：7 个已知投毒模型、3 个 clean baseline、7 个 strict-pass purified 模型。它不是用户文本里的完整 11 投毒模型全集，而是覆盖 v2/v3/v4/b1/b2/b3/b4 的 7-family strict-pass 净化包。
