# 三类 PPE 净化七攻击准入证据复核记录

日期：2026-06-02

## 背景

本轮在 `D:\联合防御模块\purification_lab` 复核三类 PPE 投毒、攻击视频、净化训练与准入证据闭环，不修改 `model/src` 主检测链路。类别顺序固定为 `0=helmet, 1=head, 2=person`，依据为 `purification_lab/configs/lab_config.yaml` 与各 v2strong 数据集 `data.yaml`。

本记录以当前文件和命令结果为准，正式准入范围只采用 `helmet_v2strong` 的 `triplet_*_universal_sandwich_detox.json`，早期 `helmet_v1` 与 `*_clean_poisoned*` 仅作为历史诊断材料，不作为本轮准入结论。

## 当前判断

已证明：

- 7 类 v2strong 攻击均具备 poisoned dataset、`attack_eval/manifest.jsonl`、poisoned checkpoint、`universal_sandwich_detox` purified checkpoint、攻击视频、3 秒抽帧记录、triplet 评估报告和 sha256 覆盖。
- `triplet_matrix.json/csv` 已重新聚合 7 个 v2strong 官方 triplet 报告，共 21 行 clean/poisoned/purified 评估记录。
- 7 个 purified checkpoint 的净化 gate 通过：purified ASR 均不高于 0.10，mAP50-95 drop 与 helmet/head/person per-class drop 均未超过配置阈值。
- 7 个攻击视频均已抽取连续 3 秒、36 帧，并通过 contact sheet 逐帧人工查看；未发现黑帧、缺帧、资源缺失、明显错序或误导性显示。
- `pre_integration_package.md/json`、`admission_gate_report.json`、`sha256_manifest.json` 与 `triplet_matrix.json/csv` 已统一到 v2strong 证据，不再保留 v1 推荐项。

失败或边界：

- `oda_warp_lowfreq` 的视频验收通过，purified ASR 为 0.0889，也满足净化 gate；但其 clean ASR 为 0.7944，说明当前 ASR 判据下 clean baseline 已对该攻击样本高触发，不能证明该攻击构造是 clean-low / poisoned-high 的强攻击。
- 因 `oda_warp_lowfreq` 攻击有效性未通过，本轮总体状态仍为 `pre_integration_ready_with_attack_limitations`，不能声明七攻击全部强生效。
- 目前结论证明的是本轮 v2strong 矩阵下 purified checkpoint 可作为下一轮接入评审材料；对重新设计后的更强 ODA warp 攻击，防御边界仍待实验确认。

待实验确认：

- `oda_warp_lowfreq` 需要重新审查 ASR 成功判据、触发 ROI、manifest 构造和 clean baseline 响应，确认是否是攻击定义过宽、触发样本天然命中，还是评估口径不适合 warp 类 ODA。
- 若后续要求“七攻击全部强生效”，应单独重建 `oda_warp_lowfreq` 攻击设计并重训 poisoned/purified checkpoint；不能把当前 pass 的净化 gate 等同于强攻击证明。
- 将 purified checkpoint 接入主检测链路后的实时显示、证据页展示和回滚流程仍需下一轮在 `model/src` 中单独验证。

## 接入建议

本轮不修改 `model/src`。下一轮最小接入建议：

- 在 `model/src/defense/runtime/catalog.py` 注册 purified checkpoint 候选、sha256 和证据路径。
- 在 `model/src/defense/runtime/evidence.py` 增加或索引净化证据包路径。
- 在 `model/src/defense/model_security/registry.py` 与 `model/src/defense/model_security/service.py` 挂接信任 registry、准入状态和回滚元数据。
- 在 `model/src/defense/web/static/model_security.html` 展示 triplet matrix、gate 结果、攻击视频、抽帧验收和 sha256。
- 回滚方案：保留 baseline_yolov8/baseline_yolov5 catalog 选择，发现异常时切回 baseline，不修改检测后处理语义、阈值或 PPE 语义。

## 结论

本轮已达到“v2strong 七攻击净化候选可进入预接入评审”的状态：证据齐全、视频验收齐全、sha 覆盖齐全、净化 gate 通过。唯一限制是 `oda_warp_lowfreq` 不能证明为 clean-low / poisoned-high 强攻击，因此总状态应保持 `pre_integration_ready_with_attack_limitations`，后续如需七攻击全部强生效，应优先重做 `oda_warp_lowfreq` 攻击设计与评估口径。
